Object类型

创建方式

  1. 使用构造函数

  2. 使用对象字面量表示法

    对象的属性名会自动转换成字符串。

    使用字面量创建对象，实际上不会调用object构造函数。（FF2及更早回调用，FF3之后不会）

    此种方法，常用于函数传參。对个参数时更加灵活，使用typeof判断每个属性是否存在。

Array类型

特点：

  1. 一个Array数组可以存储不同类型的数据。
  2. 数组的大小可以动态调整。

创建方式

  1. 使用构造函数

    可以传递数组的长度，使其自动生成指定长度的数组。

    可以直接传递数组的内容。

    new操作符可以省略，结果与之前相同。

  2. 数组字面量

lenght属性

数组length属性可以返回数组的长度。

注意：

  • length属性不是只读的，可以通过设置length属性扩展数组的长度。超出数值设置为undefined。
  • 下标访问可以超出数组的长度，返回undefined。
  • 利用length属性可以方便的数组尾部添加新项。

检测数组

  1. instanceof

    问：基于单一执行环境，当存在两个不同的全局执行环境，则存在两个以上不同版本的Array构造函数。（不使用于框架）

  2. ES5新增Array.isArray()判断是否是数组。

    支持浏览器：IE9+，FF4，Safari5+，Opera10.5，chrome

  3. Object.prototype.toString.call()

转换方法

  1. 使用toString(),toLocalString()会对数组中每一项调用toString()其返回的字符串使用逗号分解成一整个字符串返回。

    两者方法基本返回相同的值，但不总是如此，可以自己进行定义。

  2. 使用join()可以将数组按照指定的连接符转换成字符串。

    如果数组中存在null或undefined。则返回以空字符串表示。

    如果没有传入join连接符或者传入undefined，则使用逗号作为分隔符。IE7之前会错误以undefined为连接符。

  3. ==valueOf==

栈，队列方法

arr.push()向数组末端添加项（一项或多项），返回数组长度

arr.pop()从数组末端移除一项，返回移除项

arr.shift()从数组首端移除一项，返回移除项

arr.unshift()向数组首端添加项（一项或多项），返回数组长度

重排序方法

  1. reverse 反转数组

  2. sort排序

    默认按照升序排列，该方法会调用每个数组项的toString()转型方法，然后比较得到的字符串，已确定如何排序。

    因此，sort可以接受一个比较函数作为参数。

    • 第一个参位于第二个参数之前，返回负数
    • 第一个参和第二个参数相等，返回0
    • 第一个参位于第二个参数之后，返回正数

操作方法

concat

基于当前数组创建一个副本，不接收或接收多个参数，并把传入的参数依次添加到副本的尾部。

返回当前数组的副本，不改变数组本身。

slice

接收一个或多个参数，指定返回项的开始和结束位置（不包括结束位置）。只有一个参数情况下，返回从指定位置到末尾所有项。

不改变数组本身。

注意：

  • 参数重有负数，则利用数组长度转换为正数。

    例如：length=5，slice(-2,1) = slice(3,4)

  • 结束位置小于起始位置，返回空数组。

splice

向数组指定位置插入，删除，替换指定元素。

返回原始数组，在原数组上修改。

  • 删除：两个参数。arr.splice(删除位置下标, 删除的项数);
  • 插入：三个参数。arr.splice(起始位置, 0(删除项数), 插入项);
  • 替换：三个参数。arr.splice(起始位置, 删除项数, 插入项);向数组中先删除，在插入。

位置方法

ES5中出现。支持IE9+，FF2+，safari3+，Opera9.5，chrome

indexOf

arr.indexOf(要查找的项, (可选/默认0)查找起点的位置);

从数组指定位置开始向后查找，注意使用全等操作符比较。

未找到返回-1。

lastIndexOf

arr.lastIndexOf(要查找的项, (可选/默认末尾)查找起点的位置);

从数组指定位置开始向前查找，注意使用全等操作符比较。

未找到返回-1。

迭代方法

ES5增加5个方法，均接受两个参数：在每一项上运行的函数和（可选）运行该函数的作用域对象。

传入这些方法中的函数会接受三个参数：数组项的值，该项在数组中的位置，数组对象本身。

支持：IE9+，FF2，Safari3+，Opera9.5，chrome

every

在数组中的每一项运行给定函数，如果每一项都返回true，则返回true。如果有一项是false则直接返回false，不在运行。

filter

对数组中每一项运行给定函数，返回该函数会返回true项组成的数组。

forEach

对数组的每一项运行给给定函数。没有返回值。

map

对数组中每一行运行给定函数，返回每次函数调用的结果组成的数组。

some

对数组中每一项运行指定函数，如果该数组任一项返回true，则返回true。当有一项返回true时，直接返回true，下面的不在进行计算。

缩小方法

ES5增加reduce和reduceRight。迭代数组所有项，构建一个最终返回值。

接收参数两个：一个在每一项上调用的函数和（可选）作为缩小基数基础 的初始值。

调用函数接受四个参数：前一个值，当前值，项的索引，数组对象。这个函数返回的任何值都会作为第一个参数自动传递给下一项。注意：第一次迭代发生在数组第二项上，第一个参数是数组第一项，第二个参数是数组第二项。

支持：IE9+，FF3+，Safari4+，Opera10.5，chrome

reduce

从数组第一项开始，逐个遍历到最后。

reduceRight

从数组的最后一项开始，向前遍历到第一项。

方法总结

  • 堆栈方法：push，pop，shift，unshift
  • 操作方法：concat，slice，splice，join
  • 位置方法：indexOf，lastIndexOf
  • 迭代方法：every，some，filter，map，forEach
  • 缩小方法：reduce，reduceRight

注意：

  • reduce，reduceRight适用于查找数组最大／小值，累加等
  • map适用于对数组每一项执行相同的操作，例如求取每一项的平方。
  • forEach适用于利用数组执行循环操作。
  • some适用于判断数组是否含有某一项
  • every适用于判断数组是否都满足条件

javascript中的队列和栈

发表于 2018-06-23 | 分类于 javaScript |

队列

队列是一种特殊的线性结构，只允许在队列的首部进行删除操作（出队），而在队列的尾部进行插入操作（入队）。符合先进先出（First In First Out，FIFO）原则。

在javascript中，使用array.push()和array.shift()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

  • 小游戏：揭秘数字

规则:首先将第 1 个数删除，紧接着将第 2 个数放到 这串数的 尾，再将第 3 个数删除并将第 4 个数放到这串数的 尾，再将第 5 个数删除…… 直到剩下最后一个数，将最后一个数也删除。按照刚才删除的顺序，把这些删除的数连在一起就是需要的数字。数组：“6 3 1 7 5 8 9 2 4”

栈

栈是一种后进后出（Last In First Out，LIFO）的数据结构。栈限定了只能在一端（栈的顶部）进行插入和删除的操作。

在javascript中，使用array.push()和array.pop()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.pop()：从数组中移除末项并返回移除项。

  • 小游戏：验证回文
  • 小游戏：验证括号匹配

思路： 利用栈的后进后出规则，但遇到左括号末端添加，遇到匹配右括号末端推出。

  • 纸牌游戏

规则：将一副扑克牌平均分成两份，每人拿一份。一人先拿出手中的 第一张扑克牌放在桌上，然后另一人也拿出手中的第一张扑克牌，并放在刚打出的扑克牌 的上面，就像这样两人交替出牌。出牌时，如果某人打出的牌与桌上某张牌的牌面相同，即可将两张相同的牌及其中间所夹的牌全部取走，并依次放到自己手中牌的 尾。当任意一人手中的牌全部出完时，游戏结束，获胜。

javascript中插入删除数组总结：

array.pop()：从数组中移除末项并返回移除项。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

array.unshift()：向数组的第一项添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

javascript中的正则表达式

发表于 2018-06-22 | 分类于 javaScript |

ECMAScript通过RegExp数据类型来支持正则表达式。

字面量形式创建

var expression = / pattern / flags ;

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = /.at/gi;

  • pattern：

    任何简单或复杂的正则表达式。可包含字符类，限定符，分组，向前查找以及反向引用。

  • flags：标明正则表达式的行为，支持三个标志。
    • g：表示全局模式，即模式将被应用于所有字符串，而非发现第一个匹配时立即停止。
    • i：表示不区分大小写模式。
    • m：表示多行模式，即在一行文本末尾还会继续查找下一行是否存在与模式匹配的项。

使用RegExp构造函数创建

接收两个参数，第一个要匹配的字符串模式，第二个标志字符串。

var expression = new RegExp( "pattern", "flags" );

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = new RegExp(".at", "gi");

注意：双转义

使用构造函数传入必须为字符串，在某些情况下要对字符进行双重转义。

下面给出部分例子：

字面量模式 等价的字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/age/ "name\\\/age"
/\d.\d{1,2}/ "\\d.\\d{1,2}"
/\w\\hello\\123/ "\\w\\\\hello\\\\123"

在ES3中，正则表达式字面量始终为共享同一个RegExp实例，而使用构造函数创建的每一个新RegExp都是一个新的实例。

在ES5中，明确规定字面量模式也必须创建新的实例。IE9，Firefox4+，Chrome做出了修改。

常用正则表达基本概念

pattern可包含内容：

  • 普通字符：没有显示指定为元字符的所有可打印不可打印字符。

    • 非打印字符，需要进行转义。

  • 特殊字符

    具有特殊含义的字符，包括元字符。需要进行转义。

    使用\进行转义，例如我们需要匹配a?b则应该写成var exp = /a\?b/g;

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | \d | 匹配一个数字字符，等价于[0-9] |
    | \D | 匹配一个非数字字符，等价于[^0-9] |
    | \s | 匹配任何空白字符，包括空格，制表符，换页符等。等价于[\f\n\r\t\v] |
    | \S | 匹配任何非空白字符。等价于[^\f\n\r\t\v] |
    | \w | 匹配字母，数字，下划线。等价于[A-Za-z0-9_] |
    | \w | 匹配非字母，数字，下划线。等价于[^A-Za-z0-9_] |

  • 限定符

    指定正则表达式中一个给定组件出现的次数。

    | 字符 | 描述 |
    | —– | —————– |
    | * | 0或多次 |
    | + | 1或多次 |
    | ？ | 0或1 |
    | {n} | n次 |
    | {n,} | 大于等于n次 |
    | {n,m} | n-m次（包含n，m） |

    贪心模式和非贪心模式转换

    *,+都是贪心模式，尽可能匹配多的字符串，当在*,+后面放置?可以将贪心模式转换为非贪心模式（最小匹配）。

  • 定位符

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | ^ | 匹配输入字符串开始的位置。如果设置了m模式，还会与\n,\r之后的位置匹配。 |
    | $ | 匹配输入字符串结束的位置。如果设置了m模式，还会与\n,\r之前的位置匹配。 |
    | \b | 匹配一个字边界，即字与空格间的位置。 |
    | \B | 非字边界匹配。 |

  • 选择

    (选择1 | 选择2 | 选择3)

    圆括号副作用会使相关匹配被缓存，可使用?:消除缓存?:(选择1 | 选择2 | 选择3)

运算符优先级

从左到右进行计算，训训有限即规则

这次主要记录一下冒泡排序和快速排序～

冒泡排序

  • 基本思想：

    每次比较两个相邻的元素，如果它们的顺序错误就把它们交换过来。

  • 总结:

    如果有 n 个数进行排序，只需将 n-1 个数归位，也就是说要进行n-1 趟操作。而“每一趟”都需要从第 1 位开始进行相邻两个数的比较，将较小的一个数放在后面，比较完毕后向后挪一位继续比较下面两个相邻数的大小，重复此步骤，直到最后一个尚未归位的数，已经归位的数则无需再进行比较。

    O(N^2)

快速排序

  • 原理：

    为了方便记录以数组arr = [6, 1, 2, 7, 9, 3, 4, 5, 10, 8]为例。

    1. 首先选取一个基数。这里选取第一个数值。

      base=arr[0]=6;

    2. 从数组最右边开始（j = arr.length）向左移动寻找小于基数的数值。

      j=9开始，在j=7， arr[7]=5停止。

    3. 从数组最左边开始（j=0）向最右边移动寻找大于基数的数值。

      i=0开始，在i=3， arr[3]=7停止。

    4. 当i,j不相等时，交换i,j所对应的数值。重复2，3步骤。

      [6, 1, 2, 5, 9, 3, 4, 7, 10, 8]

      重复2，3步骤，继续查找。在j=6，arr[6]=4，i=4，arr[i]=9停止。

      [6, 1, 2, 5, 4 , 3, 9 , 7, 10, 8]

    5. 当ij相等时，交换基数和i所对应数值。

      继续步骤2，当j=5时，j停止

      继续步骤3，当i=5时，i==j

      当i==j时，交换base和i的数值。

      [3, 1, 2, 5, 4 , 6, 9 , 7, 10, 8]

    6. 交换基数后，在基数左边的数全部小于基数，基数右边的数全部大于基数。完成了第一次排序。接下来，需要对基数左边和基数右边的书分别重复上面操作，依次迭代。

      左边[3,1,2,5,4]，右边[9,7,18,8]分别执行步骤1-5。直到基数左边和右边仅有一个数值，不在需要进行排序。

  • 时间复杂度

    O (NlogN) 平均时间复杂度

BFC

BFC 即Box Formatting Context的缩写，是一种css样式渲染的方式，可以简单理解为在BFC环境内的元素将按照BFC方式进行渲染。

最常见的 Formatting context 有 Block fomatting context (简称BFC)和 Inline formatting context (简称IFC)。CSS3 中还增加了 GFC和 FFC。

BFC(Block formatting context)直译为”块级格式化上下文”。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

BFC布局规则：

  1. 内部的块元素会换行排列。
  2. 属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠。
  3. 每个元素默认与父容器border-left对齐，即使存在浮动也是如此。
  4. BFC的区域不会与float元素区域重叠。（注意： 此处float元素不包含在这个不重叠BFC中，即不是子元素）
  5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  6. 计算BFC的高度时，浮动元素也参与计算。（注意：默认（不是BFC）不会计算，会导致盒子塌陷）

BFC触发条件：

  1. 根元素
  2. float属性不为none
  3. position为absolute或fixed
  4. display为inline-block, table-cell, table-caption, flex, inline-flex
  5. overflow不为visible

应用：

  1. 浮动盒子塌陷问题（清除浮动）

    应用BFC布局原则6：计算BFC的高度时，浮动元素也参与计算。

    ​

  1. float文字环绕问题

    应用BFC布局原则4：BFC的区域不会与float元素区域重叠。。

  2. 防止垂直 margin 重叠

    应用BFC布局原则2／5：属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠；BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。

grid基本概念

grid布局是一种二维布局样式。

容器元素属性

display: grid;

通过display属性设置属性值为grid或inline-grid可以创建一个网格容器。网格容器中的所有子元素就会自动变成网格项目（grid item）。

设置网格行列

grid-template-columns,grid-template-rows

取值：

  • none， 默认值。
  • 明确数值，grid-template-columns: 40px 500px;
  • fr，弹性长度。类似于flex中的flex: 1;。grid-template-columns: 1fr 1fr 2fr将总宽度分为4分，分别占1/4，1/4，1/2。grid-template-columns: 400px 1fr总宽度-400px剩余宽度。
  • %，grid-template-columns: 20% 80%
  • minmax()，设置网格最小最大尺寸。grid-template-rows: minmax(100px, auto);最小100px，最大宽度自适应。
  • repeat()，设置重复尺寸。grid-template-columns: 30px repeat(2, 1fr) 30px;等同于grid-template-columns: 30px 1fr 1fr 30px;

例子：item会按照划分的格子依次进行填充

设置网格间距

grid-column-gap,grid-row-gap,grid-gap

创建列与列(行与行)之间的间距。grid-gap只能创建列与列或行与行之间的间距，但不能创建列和行与网格容器边缘的间距。

grid-gap: grid-row-gap grid-column-gap | grid-row-column-gap;

  • 默认值为0
  • 间距(Gap)可以设置任何非负值，长度值可以是px、%、em等单位值。

设置网格流向grid-auto-flow

网格默认流方向是row，可以通过grid-auto-flow属性把网格流的方向改变成column　

grid-auto-flow: column

设置网格对齐

justify-items,align-items应用与网格容器上

justify-self,align-self 应用与网格项目上，单独设置网格对齐方式。

基础概念注意问题

XHTML与HTML区别

  • XHTML标签名必须小写，标签必须严格嵌套，标签（包括空元素）必须封闭。
  • XHTML属性名必须小写，属性值必须用双引号括起来，属性值必须使用完整形式。
  • XHTML区分内容标签和结构标签。

在HTML中引入CSS的方式

方式分类

  • 行内式：在标签的style属性中设置样式

  • 嵌入式：在heade中style标签內添加样式

  • 导入式和链接式：将独立CSS文件引入到HTML中

    • 链接式：

      <link rel="stylesheet" href="style.css">

    • 导入式：

区别

  • 行内式：优先级高于其它方式，但不利于复用和维护，不推荐使用。
  • 嵌入式：针对单一页面方便，多页面网站复用性差。
  • 链接式：在装载页面主体之前装载CSS样式，页面加载出来就带有样式。引入多个CSS文件，调整CSS分类需同时调整HTML文件。
  • 导入式：在整个页面装载完成后装载CSS样式，网页文件体积较大时会出现抖动。注意使用导入式应将@import语句写在style的开始位置，即其它样式之前。
  • 建议：当引入多个CSS文件时，可以使用链接式引入一个总CSS文件，在总CSS文件中使用导入式引入其它CSS文件。

CSS继承性（inheritance）与层叠性

CSS继承性是指自子标签是否能继承父标签的所有样式，并可以在父标签样式的基础上进行修改。利用继承关系可以大大缩小代码编写量，并提高可读性。

CSS层叠性可以理解为CSS解决‘’冲突“的方式。

flex布局

发表于 2018-05-29 | 分类于 css |

flex即(Flexible Box)的缩写，意思为弹性盒模型。2009年由W3C提出的一种新的布局方案，目前已得到所有浏览器支持。

注意：

  • 任何元素都可以指定为flex布局，包括行内元素（inline-flex）。
  • 子元素float，clear，vertical-align属性失效。

flex基本概念

可以把flex元素想象成一个容器，容器中所有子元素自动成为容器成员（flex item）。

flex容器默认存在两根轴：主轴（main-axis，默认水平），垂直交叉轴（cross-axis，与主轴垂直。）

  • flex-start：主轴起始点（main start）或交叉轴起始点（cross start）
  • flex-end：主轴终点（main start）或交叉轴终点（cross end）
  • main size：每个容器成员（子元素）沿主轴的大小。
  • cross size： 每个容器成员（子元素）沿交叉轴的大小。

容器元素设置属性

  • diaplay: flex;
  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content

首先我们必须对容器元素设置display: flex; 设置弹性盒模型diaplay: flex;使其成为弹性盒模型。然后对其属性进行进一步设置。

flex-direction

设置主轴方向，即项目的排列方向。

  • row（默认值）：主轴为水平方向，起点在左端。
  • row-reverse：主轴为水平方向，起点在右端。
  • column：主轴为垂直方向，起点在上沿。
  • column-reverse：主轴为垂直方向，起点在下沿。

下图分别为column-reverse，column，row，row-reverse

flex-wrap

设置容器项目在沿主轴排列不下时，是否进行换行。默认不进行换行，当宽度超出时会进行挤压（挤压到内部元素撑开最小宽度），但border不会进行挤压。

  • nowrap（默认值）：不换行。当宽度超出时会进行挤压（挤压到内部元素撑开最小宽度），但border不会进行挤压。
  • wrap：当一行排列不下时，进行换行。第一行在上方。
  • wrap-reverse：当一行排列不下时，进行换行。第一行在下方。

下图分别为nowrap，wrap，wrap-reverse

flex-flow

flex-flow是flex-direction和flex-wrap的缩写。默认值时row nowrap。

justify-content

设置容器项目（flex item）沿主轴的对齐分布方式。

  • flex-start（默认值）：左对齐
  • flex-end：右对齐
  • center： 居中
  • space-between：两端对齐，项目之间的间隔都相等。
  • space-around：每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍。

align-items

设置容器项目（flex item）沿交叉轴的对齐分布方式。

  • flex-start：交叉轴的起点对齐。
  • flex-end：交叉轴的终点对齐。
  • center：交叉轴的中点对齐。
  • baseline: 项目的第一行文字的基线对齐。
  • stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。

下图分别为flex-start，flex-end，center，baseline，stretch。

align-content

属性定义了交叉线的对齐方式。如果项目只有一根轴线（即没有进行换行），该属性不起作用。

  • flex-start：与交叉轴的起点对齐。
  • flex-end：与交叉轴的终点对齐。
  • center：与交叉轴的中点对齐。
  • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
  • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
  • stretch（默认值）：轴线占满整个交叉轴。

容器项目（flex item）设置属性

  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex
  • align-self

对于每一个容器项目可以设置它的属性。

order

定义项目的排列顺序。数值越小，排列越靠前，默认为0。

可以设置每一个容器项目的order，不同order情况下，order越大排列越靠后。相同order情况下，按照容器项目本身顺序进行排序。

flex-grow

定义容器项目的放大比例，默认为0，即如果存在剩余空间，也不放大。

当设置flex-grow，且存在剩余空间时，该容器项目会进行放大，占用剩余空间。

当所有项目flex-grow相同时，平分剩余空间。当项目flex-grow不同时，按照分数进行分配。

flex-shrink

定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。

如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。负值无效。

flex-basis

定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。

与设置容器项目width／height相同，当超出时会被压缩。

flex

属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选。

该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。

建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值。

align-self

属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch

注意

  • 当容器项目设置绝对定位会使其脱离文档流，不受父元素控制，也会影响父元素子集其他元素分布。

绝对定位会使其不脱离文档流，但是位置可以进行进一步精确。

  • 当父元素宽高无法定死，很难预料其分布。
