浅读《图解HTTP》，这本书有一点老了，信息跟不上现在更新的步伐。不过作为了解的入门还是非常浅显易懂的。这也算不上什么笔记，知识对书中认为重要的知识点做了一个小的概括。

web及网络基础

TCP/IP

把与互联网关联的协议结合起来称为TCP/IP。

TCP/IP分层

TCP/IP是分层的，进行网络通信时发送端从应用层往下走，接收端则往应用层上走。

  • 应用层 FTP、DNS、HTTP

    决定了向用户提供应用服务时通信的活动。

  • 传输层 TCP、UDP

    对上层应用层，提供处于网络连接中两台计算机之间的数据传输。

    进行数据分割，并在报文上标记序号及端口号

  • 网络层 IP

    处理在网络上流动的数据包，规定传输路线。

    增加MAC地址转发给链路层

  • 链路层

    处理连接网络的硬件部分。硬件上的范畴均在链路层的作用范围。

IP

位于网络层，作用就是把各种数据包传送给对方。

IP地址指明节点被分配地址，可以和MAC地址进配对。

TCP

位于传输层，提供可靠的字节流服务。及把大数据分割为报文并准确送达对方。

TCP如何确保准确送达目的地：

TCP三次握手

  1. 发送端发送一个带SYN标志数据包给对方
  2. 接受端收到后回传一个SYN/ACK标志数据包确认信息
  3. 发送端回传一个带ACK标志数据包，代表握手结束建立连接。

TCP握手阶段莫名中断，会以相同顺序放相同数据包。

DNS

Domain Name System，位于应用层，提供域名到IP地址之间的解析服务或逆向查询。

URI、URL

URI：统一资源标识符，用于标识互联网资源

URL：表示资源的地点。是URI的子集。

浏览器打开页面过程

  1. 在地址栏输入URL回车
  2. 发送给DNS域名，查询对应IP地址。
  3. HTTP协议生成针对WEB服务器的HTTP请求报文
  4. TCP将请求报文分为报文段发送
  5. IP协议负责网络传送线路
  6. 接收方TCP接收到报文段重组位报文
  7. 接收方HTTP对web服务器请求内容进行处理

简单HTTP协议

http协议默认80端口；https默认使用443端口

http：HyperText Transfeer Protocol，超文本传输协议。

特点

  • 无状态：不做持久化处理。

    解决无状态引入cookie技术

  • 持久连接 HTTP1.1开始默认持久连接

  • 请求必须由客服端发出，服务端回复响应。

缺点

  • 通信使用明文 窃听
  • 不验证通信方身份 伪装
  • 无法证明报文完整性 篡改

HTTP方法

  • GET获取资源
  • POST传输实体主体
  • PUT传输文件，不带验证机制存在安全问题
  • HEAD获得报文首部，用于确定URI有效性及资源更新的日期等
  • DELETE删除文件，不带验证机制
  • OPTIONS询问支持方法
  • TRACE追踪路径，容易引发XST跨站追踪攻击
  • CONNECT要求用隧道协议连接代理，主要使用SSL、TLS

持久连接 （HTTP1.1默认持久连接）

只要任意一端没有明确提出断开连接，则保持TCP连接状态。

好处：减少TCP连接的重复建立和断开造成的额外开销，减轻服务器荷载。提高web页面显示速度。

管线化

不用等待响应亦可发送下一个请求。并发发送请求。

Cookie状态管理

通过在请求和响应报文中写入Cookie信息控制客户端状态。

HTTP信息

HTTP报文大致分为：报文首部+ 空行+报文主体。

请求报文

报文首部 + 空行 + 报文主体

  • 报文首部：请求行+请求首部字段+通用首部字段+实体首部字段+其他
  • 请求行：请求方法，请求URI，协议版本

响应报文

报文首部 + 空行 + 报文主体

  • 报文首部： 状态行+响应首部字段+通用首部字段+实体首部字段+其他
  • 状态行： 协议版本，状态吗，解释状态码的原因短语

Cookie相关首部字段

  • set-Cookie 响应首部字段

    expires：有效期（默认浏览器关闭前为止）

    domain：域名（默认创建Cookie的服务器域名）

    HttpOnly：加以限制，使Cookie不能被js脚本访问，防止跨站脚本攻击（XSS）

    Secure：仅在HTTPS安全通信发送Cookie

  • Cookie 请求首部字段

发送方式和格式

  1. 压缩传输内容编码

  2. 分割发送的分块传输编码

  3. 发送多种数据的多部分对象集合：MIME

    在HTTP首部字段加Content-type

    • multipart/form-data：web表单文件上传
    • multipart/byteranges：状态码206相应报文包含多个范围的内容

  4. 获取部分内容的范围请求：

    请求头部加Range，并且首部字段Content-Type标明multipart/byteranges

    响应返回状态码206，无法响应范围请求则返回全部和200

  5. 内容协商返回最合适内容（Content Negotiation）

媒体类型

    • text/html, text/plain, text/css

  • 图片文件

  • 应用程序使用的二进制文件

HTTP状态码

负责表示客服端HTTP请求的返回结果，标记服务器端的处理是否正常，通知出现的错误等工作。

状态码以3位数字和原因短语组成，第一位指定响应类别。

状态码类别

  • 1XX 接受的请求正在处理 信息性状态码
  • 2XX 请求正常处理完毕 成功状态码
    • 200 OK
    • 204 No Content 返回响应报文不含实体的主体部分
    • 206 Partial Content 范围请求
  • 3XX 需要进行附加操作以完成请求 重定向状态码
    • 301 Moved Permanently 永久性重定向
    • 302 Found 临时重定向 （如保存在书签中，不会修改书签保存地址） POST变成GET
    • 304 Not Modified 客服端发送附带条件请求时服务端逊于请求访问资源单未满足条件（返回不包含任何响应主体部分）。
    • 307 Temporary Redirect 临时重定向。禁止POST变成GET
  • 4XX 无腹鳍无法处理请求 客户端错误状态码
    • 400 Bad Request 请求报文存在语法错误
    • 401 Unauthorized 需要认证信息或认证失败
    • 403 Forbidden 请求被拒绝，没有资源访问权
    • 404 Not Found 未找到资源
  • 5XX 服务器处理请求出错 服务器错误状态码
    • 500 Internal Server Error 服务端执行请求发生错误
    • 503 Service Unavailable 服务器暂时处于超负荷或停机维护

Web服务器

使用单台虚拟主机实现多个域名：（Web托管服务）IP地址相同，根据域名host区分。

通信数据转发程序：代理、网关、隧道

将请求转发给通信线路上的下一站服务器，并且能接受从那台服务器发送的响应再转发给客户端。

代理

有转发功能的应用程序，扮演服务器和客户端中间人角色为两者传递信息。

  • 代理不改变请求URI，会直接发送给前方目标服务器。
  • 每次通过代理服务器，会追加Via首部信息

使用代理原因：利用缓存技术减少网络带宽的流量，组织内部针对特定网站的访问控制，以获取访问日志位主要目的等。

代理方法：缓存代理，透明代理

网关

转发其他服务器通信数据的服务器。

  • 利用网关可以由HTTP请求转化为其他协议通信（区别于代理）
  • 提高通信的安全性。客户端和网关之间可以加密

隧道

在客户端和服务端之间进行中转，并保持双发通信连接的应用程序。使用SSL等加密手段进行通信。

目的：确保客户端能与服务器进行安全的通信，隧道本身不会解析HTTP请求。

缓存

指代理服务器或客户端本地磁盘保存的资源副本。

优点：减少对资源服务器的访问，节省了通信流量和通信时间。

缓存服务器优点：避免多次从原服务器转发资源，客户端就近获取资源。

HTTPS

解决http存在部分缺陷，在http上加入加密处理和认证机制即https。HTTPS采用混合加密机制（共享密钥和公开密钥加密混用）。

与HTTP区别

通信接口部分使用SSL和TLS协议代替。通常HTTP直接和TCP通信，改为先和SSL通信，在再由SSL和TCP通信。使HTTPS拥有加密，证书和完整性保护功能。

为什么不一致使用HTTPS

  1. 加密通信消耗更多CPU及内存资源
  2. 购买证书需要开销

SSL

加密方式

采用公开密钥加密，公开密钥加密使用非对称密钥（私有密钥，公开密钥）。

发送密文的一方使用对方的公开密钥进行加密处理，对方收到加密信息后使用自己的私有密钥进行解密。

如何证明公开密钥的正确性：数字证书

解决方案：使用SSL加速器硬件改善。

ES6扩展—函数

参数默认值

规则

  • 在ES6中允许直接为函数参数设置默认值。

    在ES5中不能直接为函数参数指定默认值，必须使用||运算进行赋值，并且传入参数是false需要特别判断。

  • 参数变量是默认声明，不能使用let或const再次声明。

  • 使用默认参数时，函数不能有同名参数。

  • 参数默认值是惰性求值，只有在使用时进行计算。

  • 可以和解构赋值默认结合使用

    注意：{}很重要，当没有传入参数时，默认解析{}

  • 参数默认值的位置应该是尾参数

    非为参数设置默认值，实际上无法省略。因为后面参数不能跳过该参数赋值。除非显示声明该参数是undefined。

  • 函数的length属性

    指定默认值后，函数的length属性将不计算该参数。

  • 函数参数作用域问题

    当参数设置默认值后，函数进行声明初始化会形成一个单独作用域，函数初始化结束后作用域消失。不设置参数默认值不会出现。

应用

  • 利用参数默认值，可以指定某一个参数不得省略，省略就会抛出错误。
  • 设置参数默认值为undefined表示参数可以省略。

rest参数

ES6引入rest参数，形式为…变量名，用于获取函数的多余参数。但是rest参数必须置于函数参数最后一个，后面不能在跟参数（否则报错）。

优点

  • 可以向函数传入任意数目的参数。

  • arguments是类数组非数组，不能直接使用数组的方法需要使用Array.prototype.slice.call先将其转换为数组。rest参数不參在转换，就是一个数组。

  • 使用函数length属性，不包括rest参数。

严格模式

从ES5开始，函数内部可以设定为严格模式。必须显示声明。

ES6直有函数使用了默认值，解构赋值，扩展运算符，那么函数内部就不能显示设定为严格模式，否则报错。

原因：严格模式设定后包括函数参数和函数内部。函数先执行参数在执行函数内部，如果显示声明严格模式，则函数参数执行时还未声明。

解决办法：方法一：全局声明严格模式；方法二：把带默认参数函数包含在一个无参数的立即执行函数里面。

name属性

返回该函数的函数名。

ES5匿名匿名函数函数表达式返回空字符串。ES6匿名函数函数表达式返回函数实际函数名。

  • Function构造函数返回的函数实例，name属性的值为anonymous。

(new Function).name // "anonymous"

  • bind返回的函数，name属性值会加上bound前缀。

(Math.sin.bind()).name; //"bound sin"

箭头函数

规则

  • 参数部分使用圆括号

  • 代码块部分使用大括号扩起来，并使用return返回。

    • 仅有一条语句可省略大括号，省略return也会返回该语句。

    • 返回为对象且仅一句，大括号不能省略

    • 仅一条语句，并且不让该语句返回则使用

      let fn = () => void doesNotReturn();

  • 可以和变量解构结合使用

  • 简化回调函数

    [1,2,3].map(x => x * x);

    var result = values.sort((a, b) => a - b);

    const headAndTail = (head, ...tail) => [head, tail];

注意

  • 函数体内部的this，时定义时所在的对象，而非使用时所在的对象。
    1. 箭头函数可以让setTimeout里面的this绑定定义时所在的作用域，而不是运行时所在的作用域。
    2. 箭头函数让this指向固定化。原因是：箭头函数内部没有this，所以只能指向外层代码块的this。
    3. 箭头函数内部没有this，arguments，super，new.target。指向外层函数对应变量。不能使用call，applay，bind。
  • 不可以当作构造函数使用。
  • 不可以使用arguments对象，可以使用rest代替。
  • 不可以使用yield命令，因为尖头函数不能作Generator函数。

双冒号运算符

双冒号的提出是为了用来取代call，apply，bind调用。

双冒号绑定绑定运算符::，双冒号左边是一个对象，右边是一个函数。该运算符会自动将左边的对象作为上下文执行环境（即this对象），绑定到右边的函数上面。

尾调用优化

尾调用（Tail Call）是函数式编程概念。指某个函数的最后一步是调用另一个函数。

ES6扩展—变量

let

ES6 新增了的声明变量方法。但是所声明的变量，只在let命令所在的代码块内有效。

特点：

  1. 存在块级作用域
  2. 未声明不得使用，不存在变量提升
  3. 不允许在同一作用域下重复声明
  4. 暂时性死区（TDZ,temporal dead zone）
  5. 使用let声明变量不会挂载载window身上。（使用var声明变量挂载在window上）

与var区别

  • let仅在代码块內有效，在代码块外使用js报错。

  • 未声明不得使用，不存在变量提升。

    对于var声明变量，在声明前使用会提升为全局变量。

    对于let声明变量，在声明前使用会报错。

    个人理解： 在预编译时，会对var，function在第一次读取时进行编译，然后在运行时进行赋值。

  • 暂时性死区

    暂时性死区的本质就是，只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到声明变量的那一行代码出现，才可以获取和使用该变量。

    • 在块作用域中，使用let声明的变量在let声明前不得使用，即使这个变量是一个全局变量。

      导致使用typeof不会绝对安全

      自相赋值报错

  • 不允许在同一个作用域下重复声明

let块级作用域

  • 外层变量不受内层变量影响

  • ES6 允许块级作用域的任意嵌套。

  • 块级作用域的出现，实际上使得获得广泛应用的立即执行函数表达式（IIFE）不再必要了。

  • 代码块內函数声明

    块级作用域内声明的函数类似于let，对作用域之外没有影响。

    ES6 的块级作用域允许声明函数的规则，只在使用大括号的情况下成立，如果没有使用大括号，就会报错。

    考虑到环境导致的行为差异太大，应该避免在块级作用域内声明函数。如果确实需要，也应该写成函数表达式，而不是函数声明语句。

  • 如果想让一个引用数据类型具有常量的特性，使用Object.freeze,冻结后非严格模式下修改不起作用，严格模式报错。

数据格式JSON

发表于 2018-06-27 | 分类于 数据格式 |

JSON是一种数据格式。虽然与js的对象具有相同的语法，但是JSON并不从属于JavaScript，很多编程语言都有针对JSON的解析器和序列化器。

语法

  • 简单值：可以在JSON中表示字符串，数值，布尔值，null，但是不支持undefined。
  • 对象：复杂数据类型，一组有序的键值对，键值可以为简单值或复杂数据类型的值。

JSON不支持变量，函数或者对象实例，可以对象，数组。

简单值

与JS区别：

  • 字符串必须使用双引号（单引号导致语法错误）。

JSON一般用来表示复杂数据，简单值只是整个数据结构一部分。

对象

与JS区别：

  • 键必须使用双引号包裹。在js中双引号单引号或没有引号都可以。
  • 在JSON中不同层级可以出现相同的属性名
  • JSON中没有变量的概念，每个键值对都表示一个数据，键值对之间用逗号连起。

数组

数组与JS中字面量数组没有区别，只需要注意没有变量的概念，必须使用双引号即可。

解析与序列化

JS可以把JSON数据结构解析为JS对象，解析为对象后操作十分方便。

JSON是javascript语法的子集，早期使用eval()函数就可以进行解析，解释并返回JS对象和数组。但是使用eval存在风险，可能被执行一些恶意代码。

ES5中定义了全局对象JSON，支持浏览器：IE8+，FF3.5+，Safari4+，Chrome，Opera10.5+，对于早期不支持原生JSON解析的浏览器，可以使用shim。

JSON对象有两个方法：stringify(),parse()

JSON.stringify()

JS对象序列化为JSON字符串，默认情况输出JSON字符串不包含任何空字符或者缩进。

序列化时所哟函数和原型成员会被忽略，不体现在结果中。此外值为undefined的任何属性也会被跳过。最终都是有效的JSON数据类型。

参数：两个参数（均可选），第一个参数为过滤器可以是数组或函数，第二个参数是一个选项表示是否在JSON字符串中保留缩进。

  • 第一个参数时数组

    表示转换JSON只包含数组中列出的属性。注意属性只能选择最外层的属性。

  • 第一个参数是函数

    传入函数接收两个参数，属性名和属性值，计算后返回属性值，若返回不符合JSON语法，则JSON忽略该键值对。

    每个键值都会循环，若出现嵌套则从左到右，从外层向内层进行。

  • 第二个参数：

    参数为整数：表示每个级别缩进的空格数。只要传入有效的参数，结果也会出现换行符，方便阅读。最大缩进数目是10，大于10的会自动转换为10。

    参数为字符串：表示使用该字符串缩进。不能超出10个字符，超出将忽略超出部分。

toJSON()

当使用JSON.stringify不能满足对对象进行自定义序列化的需求，可以都过对象上调用toJSON方法，返回自身的JSON数据格式。

原生Date对象有toJSON方法，将其转换成ISO 8601日期字符串。也可为认为对象添加toJSON方法。

JSON.parse()

把JSON字符串解析为原生JS值，若不是有效JSON抛出错误。

参数：两个参数，第一个 参数为JSON数据，第二个参数为还原函数，将在每个键值对上调用。常用于将字符串转换为相应的对象。

    3. 执行构造函数的代码
    4. 返回对象

理解原形函数

创建一个新的函数，就会为其新增加一个属性prototype，其值指向函数原型对象。函数原型对象含有instructor属性，其值指向构造函数。

注意：每个实例与构造函数的原型对象相连接，而不是构造函数。

无法访问实例的[[prototype]]属性，但是FF，safari，chrome支持属性__proto__.

判断原形函数和实例之间的关系

isPrototypeOf()

由于我们无法通过实例访问[[prototype]]属性，数亿我们无法确定实例和原形函数之间的关系，因此使用函数isPrototypeOf()

Object.getPrototypeOf()

ES5增加Object.getPrototypeOf()返回实例的[[prototype]]属性（即原型对象）。

支持：IE9+，FF3.5+，Safari5+，Opera12+，chrome

对象实例共享原型基本原理

在读取对象属性时，首先在对象实例本身中搜索属性，搜索成功返回。未搜多到结果则搜索指针指向的原型对象，搜索成功返回。未搜索到继续向原型对象搜索，直到Object则返回undefined。

原型对象中的constructor属性也是共享的。

当实例中重写了原型中的属性，则会覆盖原型中的属性，但是不影响其他实例。因为它并不会修改原型中的属性，仅会把属性添加在实例中。

不过可以使用delete删除实例的属性，让其重新访问原型中的属性。

原型的动态性

  • 可以先创建实例，再向原型中添加属性，仍然可以使用。原因：在原型中的查找过程是一次搜索

  • 创建实例后，重写整个原型对象，则切断了构造函数和最初原型之间的联系。但是已创建的实例仍指向原来的构造函数。

原生对象的原型

所有原生的引用类型（Object，Array，String，等等）都在其构造函数的原型上定义了方法。

（不推荐）可以通过修改原生引用类型的原型，添加方法。则所有该原生类型所有实例均可使用。

原型对象缺陷

不能为构造函数传递初始化参数，所有对象具有相同属性值。

  • 对于属性值为基本类型的属性，修改个别实例的属性值则会覆盖该实例的原生属性，不会产生问题。
  • 对于属性值为引用类型的属性，修改个别实例的属性值则是在修改属性值指向的引用地址的数值，在其他实例中也会产生相应变化。

具条件，一是正确性，毋庸置疑，这是必须的，二是高效性，在确保正确的前提下，如何将复杂度降到最小。

方法一

思路： 每次随机random生成一个数组的下标，取出该项。剩余数组继续随机生成下标，直到全部取出。

缺点： 当数组非常大的时候会很慢。

整体复杂度：O(n^2).

该方法中使用splice，splice的复杂度是O(n)

方法二

思路：巧妙利用Math.random。这种方法仅仅适用于非正式场合

平均复杂度：O(nlogn)

方法三： Fisher–Yates shuffle

思路：遍历数组元素，将其与之前的任意元素交换。因为遍历有从前向后和从后往前两种方式，所以该算法大致也有两个版本的实现。

复杂度：O(n)

  • 从后向前
  • 从前向后

  • 堆栈方法：push，pop，shift，unshift
  • 操作方法：concat，slice，splice，join
  • 位置方法：indexOf，lastIndexOf
  • 迭代方法：every，some，filter，map，forEach
  • 缩小方法：reduce，reduceRight

注意：

  • reduce，reduceRight适用于查找数组最大／小值，累加等
  • map适用于对数组每一项执行相同的操作，例如求取每一项的平方。
  • forEach适用于利用数组执行循环操作。
  • some适用于判断数组是否含有某一项
  • every适用于判断数组是否都满足条件
