ES6扩展—函数

参数默认值

规则

  • 在ES6中允许直接为函数参数设置默认值。

    在ES5中不能直接为函数参数指定默认值，必须使用||运算进行赋值，并且传入参数是false需要特别判断。

  • 参数变量是默认声明，不能使用let或const再次声明。

  • 使用默认参数时，函数不能有同名参数。

  • 参数默认值是惰性求值，只有在使用时进行计算。

  • 可以和解构赋值默认结合使用

    注意：{}很重要，当没有传入参数时，默认解析{}

  • 参数默认值的位置应该是尾参数

    非为参数设置默认值，实际上无法省略。因为后面参数不能跳过该参数赋值。除非显示声明该参数是undefined。

  • 函数的length属性

    指定默认值后，函数的length属性将不计算该参数。

  • 函数参数作用域问题

    当参数设置默认值后，函数进行声明初始化会形成一个单独作用域，函数初始化结束后作用域消失。不设置参数默认值不会出现。

应用

  • 利用参数默认值，可以指定某一个参数不得省略，省略就会抛出错误。
  • 设置参数默认值为undefined表示参数可以省略。

rest参数

ES6引入rest参数，形式为…变量名，用于获取函数的多余参数。但是rest参数必须置于函数参数最后一个，后面不能在跟参数（否则报错）。

优点

  • 可以向函数传入任意数目的参数。

  • arguments是类数组非数组，不能直接使用数组的方法需要使用Array.prototype.slice.call先将其转换为数组。rest参数不參在转换，就是一个数组。

  • 使用函数length属性，不包括rest参数。

严格模式

从ES5开始，函数内部可以设定为严格模式。必须显示声明。

ES6直有函数使用了默认值，解构赋值，扩展运算符，那么函数内部就不能显示设定为严格模式，否则报错。

原因：严格模式设定后包括函数参数和函数内部。函数先执行参数在执行函数内部，如果显示声明严格模式，则函数参数执行时还未声明。

解决办法：方法一：全局声明严格模式；方法二：把带默认参数函数包含在一个无参数的立即执行函数里面。

name属性

返回该函数的函数名。

ES5匿名匿名函数函数表达式返回空字符串。ES6匿名函数函数表达式返回函数实际函数名。

  • Function构造函数返回的函数实例，name属性的值为anonymous。

(new Function).name // "anonymous"

  • bind返回的函数，name属性值会加上bound前缀。

(Math.sin.bind()).name; //"bound sin"

箭头函数

规则

  • 参数部分使用圆括号

  • 代码块部分使用大括号扩起来，并使用return返回。

    • 仅有一条语句可省略大括号，省略return也会返回该语句。

    • 返回为对象且仅一句，大括号不能省略

    • 仅一条语句，并且不让该语句返回则使用

      let fn = () => void doesNotReturn();

  • 可以和变量解构结合使用

  • 简化回调函数

    [1,2,3].map(x => x * x);

    var result = values.sort((a, b) => a - b);

    const headAndTail = (head, ...tail) => [head, tail];

注意

  • 函数体内部的this，时定义时所在的对象，而非使用时所在的对象。
    1. 箭头函数可以让setTimeout里面的this绑定定义时所在的作用域，而不是运行时所在的作用域。
    2. 箭头函数让this指向固定化。原因是：箭头函数内部没有this，所以只能指向外层代码块的this。
    3. 箭头函数内部没有this，arguments，super，new.target。指向外层函数对应变量。不能使用call，applay，bind。
  • 不可以当作构造函数使用。
  • 不可以使用arguments对象，可以使用rest代替。
  • 不可以使用yield命令，因为尖头函数不能作Generator函数。

双冒号运算符

双冒号的提出是为了用来取代call，apply，bind调用。

双冒号绑定绑定运算符::，双冒号左边是一个对象，右边是一个函数。该运算符会自动将左边的对象作为上下文执行环境（即this对象），绑定到右边的函数上面。

尾调用优化

尾调用（Tail Call）是函数式编程概念。指某个函数的最后一步是调用另一个函数。

Array.from(arrayLike, item => { })

用于将类似数组的对象（array-like object）和可遍历（iterable）对象转换为真正的数组。

类似数组的对象，本质特征即必须具有length属性。任何具有length属性的对象，都可以通过Array.from方法转换为数组。扩展运算符不一定有用。

例如：Object

接收两个参数：
第一个是要转转换的内容。object，nodeList，arguments

通用写法

部署iterator接口数据结构

不管是Array.from还是扩展运算符否可以将字符串转换为数组，并且准确测量多字节Unicode字符。

Array.of

Array.of(item1,item2,item3···)

将一组值转换为数组。该方法主要用于弥补new Array构造方法传入参数个数不同会导致数组不同。

Array.of总是返回参数值组成的数组。如果没有参数，就返回一个空数组。

copyWithin

数组实例的方法，在当前数组内部，将指定start位置到end结束（不包括）之间的成员，复制到target起始的位置及之后，会修改原数组。

Array.prototype.copyWithin(target, start = 0, end = this.length)

  • target（必需）：从该位置开始替换数据。如果为负值，表示倒数。
  • start（可选）：从该位置开始读取数据，默认为 0。如果为负值，表示倒数。
  • end（可选）：到该位置前停止读取数据，默认等于数组长度。如果为负值，表示倒数。

find,findIndex

数组实例的find方法，用于找出第一个符合条件的数组成员。

[1,2,3,4,5].find( (item, index, arr) => {return item>3} )

输入参数

第一个参数是函数，选择条件返回布尔值，寻出第一个符合条件的数组成员（返回true）。回调函数可以接受三个参数，依次为当前的值、当前的位置和原数组。

第二个参数（可选）用来绑定回调函数的this对象。

find：如果没有符合条件的成员，则返回undefined。有则返回第一个符合条件成员。

findIndex：返回第一个符合条件的数组成员的位置，如果所有成员都不符合条件，则返回-1。

indexOf和findIndex对于NaN

indexOf方法无法识别数组的NaN成员，但是findIndex方法可以借助Object.is方法做到。

fill

fill方法使用给定值，填充一个数组。fill方法用于空数组的初始化非常方便。数组中已有的元素，会被全部抹去。

参数：

第一个：填充值，如果是对象是浅拷贝

第二个第三个（可选）：指定填充的起始位置和结束位置。

entries，keys，values

keys()是对键名的遍历、values()是对键值的遍历，entries()是对键值对的遍历。

与for of结合使用。活着使用遍历器对象的next方法。

includes

Array.prototype.includes方法返回一个布尔值，表示某个数组是否包含给定的值，与字符串的includes方法类似。

可选参数：第二个参数表示搜索的起始位置（可为负），当大于数组长度，则会重置为0。

可以用于判断NaN。

用于判断是否含有字符的方法，兼容方法。

数组的空位

数组的空位指，数组的某一个位置没有任何值。比如，Array构造函数返回的数组都是空位。

空位不是undefined，一个位置的值等于undefined，依然是有值的。空位是没有任何值，空位使用in不能查找到。

ES5对空位处理不一致：

  • forEach(), filter(), reduce(), every() 和some()都会跳过空位。
  • map()会跳过空位，但会保留这个值
  • join()和toString()会将空位视为undefined，而undefined和null会被处理成空字符串。

ES6 则是明确将空位转为undefined。

  • Array.from方法会将数组的空位，转为undefined
  • 扩展运算符（...）也会将空位转为undefined
  • copyWithin()会连空位一起拷贝。
  • fill()会将空位视为正常的数组位置。
  • for...of循环也会遍历空位。
  • entries()、keys()、values()、find()和findIndex()会将空位处理成undefined。

总结

  • 扩展运算符...[]：将数组转换为逗号分隔参数序列

    1. 实现深拷贝
    2. 避免使用apply
    3. 字符串转换，避免双字符

  • Array.from(arrayLike, item => { })

    将类似数组对象和可遍历对象转换为真正数组。

    1. 避免使用[].slice.call(arrayLike)
    2. 遍历非数组并且对每个元素进行操作。

  • Array.of(item1,item2,item3···)

    将一组值转换为数组。

    1. 避免new Array传入参数个数不同结果不同。

实例方法：

  • copyWithin()：数组内部复制

    Array.prototype.copyWithin(target, start = 0, end = this.length)

  • find(),findIndex()：判断数组是否具有符合条件的数组成员。并对符合元素执行操作（函数参数）

    [1,2,3,4,5].find( (item, index, arr) => {return item>3} )

  • fill()：给定数值填充数组

  • entries(),keys(),values()：与for of结合或着next结合遍历

  • includes()：判断数组是否包含给定值。

  • 数组空位问题

发表于 2018-06-29 |

Symbol类型是ES6的新增原始数据类型，是JS的第七种数据类型。

引入原因：防止对象的属性冲突，保证每个属性名都是独一无二的。

概述

Symbol函数接收参数：

注意：参数指标时当前Symbol值的描述，相同参数返回值不相等。

  • 不传参数：控制台输出都为Symbol()，不利于区分。

  • 字符串：表示对Symbol实例的描述，只要为了在控制台显示，或者转换为字符串时为较容易区分。

  • 对象：调用对象的toString方法，转换为字符串。

转换

  • 不能进行字符串拼接
  • 可以转换为字符串，布尔值
  • 不能转换为数值

应用

作为属性名

每一个Symbol值都不相等，所以其可作为标识符保证不出现同名属性。

  • Symbol值作为对象属性名时，不能使用点运算符。使用点会认为其为字符串。
  • 在对象内部使用Symbol必须防止在方括号內。
  • 可以使用==增强对象写法==

减少代码耦合

属性名遍历

使用Symbol作为属性名，不会出现在for in,in,for of,Object.keys(),Object.getOwnPropertyNames(),JSON.stringify()中。但是并不能认为其是私有属性。

  • Object.getOwnPropertySymbols()

可以获得指定对象的所有Symbol属性名。返回值为当前对象所有属性名为Symbol的值。

  • Reflect.ownKeys()

返回所有类型的键名，包括常规键名和Symbol键名。

Symbol.for(),Symbol.keyFor()

使用相同描述生成的Symbol是不相同的，有时希望，相同描述使用同一个Symbol值，这时可以使用Symbol.for(), Symbol.keyFor()。

Symbol.for()

参数：一个字符串

接收一个字符串，搜索是否有以字符串作为名称的Symbol值，如果有返回这个Symbol值，没有则创建并返回。

注意：Symbol.for为Symbol值登记的名字，时全局环境的，在不同iframe，service worker中取到同一个值。

Symbol.keyfor()

返回一个已经登记的Symbol类型值的key，返回类型为字符串或undefined（未登记）。

模块Singleton模式

Singleton模式指调用一个类，任何时候返回的都是同一个实例。

将实例传送给Symbol类型，防止外部覆盖和改写。

单例模式的思路:

一个类能返回对象一个引用(永远是同一个)和一个获得该实例的方法；当我们调用这个方法时，如果类持有的引用不为空就返回这个引用，如果类保持的引用为空就创建该类的实例并将实例的引用赋予该类保持的引用；同时我们还将该类的构造函数定义为私有方法，这样其他处的代码就无法通过调用该类的构造函数来实例化该类的对象，只有通过该类提供的静态方法来得到该类的唯一实例。

参数

ECMAScript函数不介意传递参数个数和类型，实参和形參不需要一一对应。原因是参数在内部使用一个数组表示，函数接收到的始终是一个类数组（arguments），而不关心数组中包含的参数。

arguments：

  • 接受函数传递的参数，具有数组的下标访问功能和length方法。

  • 命名参数和arguments可以混合使用，且始终保持同步。

    当命名参数没有传值的情况下，修改对应arguments不会同步。

  • 严格模式不能修改arguments。

  • 函数不具有重载功能，但是可以根据判断arguments.length来模仿实现。

callee：

arguments具有的属性，概述行是一个指针，指向拥有这个arguments对象的函数。

arguments.callee指向这个接受arguments的函数，可以后面加括号调用函数。

应用：递归调用

如果使用注释行，则函数名与函数调用耦合在一起。则console.log(fun2(5))结果为0。

严格模式下arguments.callee报错导致在严格模式下，不能使用此种方法解耦函数名和函数的调用，这里推荐使用命名函数表达式来解决这种问题，严格和非严格模式通用。

函数属性和方法

函数时对象，因此函数具有属性和方法。每个函数包含两个属性：length和prototype。

length：

函数希望接受的命名参数的个数。

prototype

对于引用类型，prototype时保存它们所有实例方法的真正所在。ES5种，prototype属性不可枚举，不能使用for-in。

每个函数都包含两个非继承而来的方法：apply(), call()

在特定的作用域中调用函数，实际上等于设置函数体内的this对象的值。

apply：

接收两个参数，一个是运行函数的作用域，另一个参数组（可以是array实例，也可以是arguments对象）

call：

接收两个参数，一个是运行函数的作用域，另一个接受函数参数，必须逐个接受，与命名参数对应。

bind：

ES5定义的方法，船舰一个函数实例，其this值被绑定到传给bind函数的值。

区别

函数内部属性

this

this引用的是函数执行的环境对象。当在网页全局作用域中调用函数时，this对象的引用就是window。

callee

arguments具有的属性，概述行是一个指针，指向拥有这个arguments对象的函数。

详见参数章节。

caller

保存调用当前函数的函数引用，如果在全局作用域中调用当前函数值为null。

严格模式下报错。

函数名.caller会指向执行函数的环境。为了解耦函数名，使用arguments.callee代替函数名。

闭包

闭包：有权访问另一个函数作用域中的变量的函数。

常见创建方式： 在一个函数内部创建另一个函数。

第一次调用函数：

  1. 创建一个执行环境

  2. 创建执行环境的相应作用域链->复制给内部属性（[Scope]）

  3. 初始化函数活动对象（使用this，arguments，其它命名参数初始化）

  4. 将函数活动对象提阿嘉岛作用域链的最前端

    外部活动对象始终处于第二位，外部函数的外部函数活动对象处于第三位，直至作为作用域链终点的全局执行环境。

  5. 无论何时访问函数都会从作用域链最前端开始搜索变量。

    一般，函数执行完毕，局部活动对象被被销毁。

    在闭包中外部函数执行完毕后，其执行环境的作用域链被销毁，但是活动对象仍保留內内存中，直到匿名函数被销毁外部函数活动对象才会被销毁。

使用闭包时，调用外部函数，返回内部匿名函数，然后对匿名函数进行调用，当解除对匿名函数的引用时才会销毁匿名函数和外部函数的活动对象。

利用闭包可以访问函数中的私有变量，利用这个原理可以创建用于访问私有变量的共有方法。把有权访问私有变量和私有函数的公有方法叫做特权方法（privileged method）。

  1. 在构造函数中定义特权方法

    使用构造函数创建特权缺点：每个实例都会创建同样一组新方法，占用内存。且必须使用构造函数模式。

    也叫稳妥构造函数模式

  1. 静态私有变量方法

    通过私有作用域定义私有变量或函数，也可以创建特权方法。

模块模式

单例（singleton）：只有一个实例的对象。

JavaScript使用对象字面量的形式创建单例对象。这种模式创建的每个单例都是Object实例。并且单例通常作为全局对象存在，不会将它传递给函数，因此没有必要使用instanceof检测对象类型。

增强模块模式

对模块模式的改进，在返回对象之前加入增强代码。

该模式适合那些单例必须是某种类型的实例，同时还必须添加某些属性和方法对其加以增强。

函数总结

  • 函数表达 vs 函数声明

    函数声明必须有函数名，会预编译

    函数表达式可以为匿名函数（拉姆达函数）

  • 递归函数

    始终使用arguments.callee递归调用自身，不使用函数名。

  • 闭包

    闭包的作用域包含自己作用域，包含函数作用域和全局作用域。

    函数返回闭包后，函数作用域将会一直在内存中知道闭包不存在为止。

  • 立即执行函数

    创建并立即调用一个函数，调用后立即销毁。除非某些变量赋值给了包含作用域中的变量。

  • 私有变量

    js中不存在私有对象属性，但是使用闭包进行模仿。

    有权访问私有变量的公有方法叫特权方法。

  • 自定义类型特权方法：构造函数模式，原型模式

    构造函数方法调用私有变量／立即执行函数中变量作用域提升

  • 单例特权方法：模块模式，增强模块模式

    即在立即执行函数中返回对象。

工厂模式

由于ECMAScript无法创建类，但是可以使用函数来封装以特定接口创建对象的细节。

优点： 创建多个相似对象，不需要重复代码。

缺点：无法解决对象识别问题（无法知道对象的类型）。

构造函数模式

构造函数可以用来创建特定类型的对象。

优点：可以识别对象类型。

缺点：不同实例上同名函数不相等，若要相等需要将函数定义为全局函数没有封装性。

使用构造函数创建的对象实例中都有一个constructor构造函数属性，该属性指向构造函数。但是我们更多实用instanceof进行判断。

解决每个实例都会创建一个方法，可使用全局函数，但是降低封装性。

  • 使用构造函数和工厂模式声明函数区别：
    1. 没有显示创建对象
    2. 直接将属性和方法赋给了this对象
    3. 没有return语句
    4. 构造函数首字母大写
  • 使用new调用构造函数创建对象过程
    1. 创建一个新对象
    2. 将构造函数的作用域赋给新对象（this指向新对象）
    3. 执行构造函数的代码
    4. 返回对象

原型模式

更简单的原型语法

上面使用Person.prototype较为复杂，减少输入，我们可以使用对象字面量重写整个原型对象。

使用这种方法重新定义的Person.prototype相当于覆盖了原有的，但是原有自动创建的Person.prototype.construtor不会在指向构造函数。尽管instanceof会返回正确结果，但是constructor无法确定对象类型。

可以在字面量对象中强行加入construtor属性指向Person。但是回到时Enumerable设置为true（可枚举）。

也可以使用Object.defineProperty

理解原形函数

创建一个新的函数，就会为其新增加一个属性prototype，其值指向函数原型对象。函数原型对象含有instructor属性，其值指向构造函数。

注意：每个实例与构造函数的原型对象相连接，而不是构造函数。

无法访问实例的[[prototype]]属性，但是FF，safari，chrome支持属性__proto__.

判断原形函数和实例之间的关系

isPrototypeOf()

由于我们无法通过实例访问[[prototype]]属性，数亿我们无法确定实例和原形函数之间的关系，因此使用函数isPrototypeOf()

Object.getPrototypeOf()

ES5增加Object.getPrototypeOf()返回实例的[[prototype]]属性（即原型对象）。

支持：IE9+，FF3.5+，Safari5+，Opera12+，chrome

对象实例共享原型基本原理

在读取对象属性时，首先在对象实例本身中搜索属性，搜索成功返回。未搜多到结果则搜索指针指向的原型对象，搜索成功返回。未搜索到继续向原型对象搜索，直到Object则返回undefined。

原型对象中的constructor属性也是共享的。

当实例中重写了原型中的属性，则会覆盖原型中的属性，但是不影响其他实例。因为它并不会修改原型中的属性，仅会把属性添加在实例中。

不过可以使用delete删除实例的属性，让其重新访问原型中的属性。

检测属性是否存在于实例／原型中

hasOwnProperty(属性名)

只有给定属性存在于对象实例中，才会返回true

getOwnPropertyDescriptor

只能用于实例属性。如果要取得原型属性的描述必须直接在原型对象上调用。

原型与in操作符

有两种方式使用in操作符：单独使用和在for-in循环中使用

单独使用in

单独使用时，in操作符会在通过对对象能够访问给定属性时返回true，无论属性存在于实例还是原型中。

  • in和hasOwnProperty一起使用确定属性存在于原型中

for-in循环

返回所有能够通过对象访问的，可枚举的（enumerated）属性，其中既包括存在于实例中的属性也包括存在于原型中的属性。

原型中不可枚举属性（即Enumerable标记的属性）在进行实例属性的for-in循环中会返回。规定：所有开发人员的定义的属性都是可以枚举的（IE8及之前例外）。

不可枚举属性和方法：hasOwnProperty，propertyIsEnumerable，toLocalString，toString，valueOf，constructor（ES5设置Enumerable=false），prototype（ES5设置Enumerable=false）。

IE早期版本bug

Object.keys

只返回实例自己的属性。

ES5增加，接收i 一个对象为参数，返回一个包含所有可枚举属性的字符串数组。

getOwnPropertyNames

返回实例自己的所有属性，包括不可枚举属性。

原型的动态性

  • 可以先创建实例，再向原型中添加属性，仍然可以使用。原因：在原型中的查找过程是一次搜索

  • 创建实例后，重写整个原型对象，则切断了构造函数和最初原型之间的联系。但是已创建的实例仍指向原来的构造函数。

原生对象的原型

所有原生的引用类型（Object，Array，String，等等）都在其构造函数的原型上定义了方法。

（不推荐）可以通过修改原生引用类型的原型，添加方法。则所有该原生类型所有实例均可使用。

原型对象缺陷

不能为构造函数传递初始化参数，所有对象具有相同属性值。

  • 对于属性值为基本类型的属性，修改个别实例的属性值则会覆盖该实例的原生属性，不会产生问题。
  • 对于属性值为引用类型的属性，修改个别实例的属性值则是在修改属性值指向的引用地址的数值，在其他实例中也会产生相应变化。

方法总结：

ConstructFun：构造函数，obj：构造函数的实例，obj = new ConstructFun();

PropertyName：属性名

  • ConstructFun.prototype 原生函数

  • ConstructFun.prototype.construtor 构造函数

  • ConstructFun.prototype.isPrototypeOf(obj)

    判断obj的原型是否为ConstructFun.prototype。返回布尔值。

  • Object.getPrototypeOf(obj)

    返回实例的原生函数。

  • obj.hasOwnProperty(PropertyName)

    判断PropertyName是否是obj实例自己的属性（非原生属性）

  • Object.getOwnPropertyDescriptor(obj, PropertyName)

    返回obj自己属性PropertyName的描述，不能返回原生属性的，

  • PropertyName in obj

    访问属性存在于obj中时返回true，无论属性存在于实例还是原型中。

  • for key in obj

    obj中所有可枚举属性（包括原生的可枚举属性）

  • Object.keys(obj)

    返回obj所有自己属性名（不包括原生属性）

  • Object.getOwnPropertyNames(obj)

    返回obj所有属性名，包括原生属性

组合使用构造函数模式和原型模式

使用最广法。使用构造函数定义实例属性，使用原型模式定义共享属性和方法。

动态原型模式

在构造函数中初始化原型（仅在必要情况下）

保持使用构造函数和原型的优点，且不改变constructor

寄生构造函数模式

不推荐使用，不做介绍

稳妥构造函数模式

没有公共属性，其方法也不引用this对象。

适合在安全环境中（环境中禁止使用this和new），或者方式数据被其他应用程序改动时使用。

使用稳妥构造函数创建的对象与构造函数之间没有关系，使用instanceof无意义。

数组乱序／洗牌问题

将一个数组中的顺序打乱有多种方法，但是哪一种最高效呢（我也不知道。。。只能记录一下我会的这些方法哪一种更好）～

推荐使用方法三

题目：给定一个数组，将其打乱，返回新的数组，即为数组乱序，也称为洗牌问题

具条件，一是正确性，毋庸置疑，这是必须的，二是高效性，在确保正确的前提下，如何将复杂度降到最小。

方法一

思路： 每次随机random生成一个数组的下标，取出该项。剩余数组继续随机生成下标，直到全部取出。

缺点： 当数组非常大的时候会很慢。

整体复杂度：O(n^2).

该方法中使用splice，splice的复杂度是O(n)

方法二

思路：巧妙利用Math.random。这种方法仅仅适用于非正式场合

平均复杂度：O(nlogn)

方法三： Fisher–Yates shuffle

思路：遍历数组元素，将其与之前的任意元素交换。因为遍历有从前向后和从后往前两种方式，所以该算法大致也有两个版本的实现。

复杂度：O(n)

  • 从后向前
  • 从前向后

    默认按照升序排列，该方法会调用每个数组项的toString()转型方法，然后比较得到的字符串，已确定如何排序。

    因此，sort可以接受一个比较函数作为参数。

    • 第一个参位于第二个参数之前，返回负数
    • 第一个参和第二个参数相等，返回0
    • 第一个参位于第二个参数之后，返回正数

操作方法

concat

基于当前数组创建一个副本，不接收或接收多个参数，并把传入的参数依次添加到副本的尾部。

返回当前数组的副本，不改变数组本身。

slice

接收一个或多个参数，指定返回项的开始和结束位置（不包括结束位置）。只有一个参数情况下，返回从指定位置到末尾所有项。

不改变数组本身。

注意：

  • 参数重有负数，则利用数组长度转换为正数。

    例如：length=5，slice(-2,1) = slice(3,4)

  • 结束位置小于起始位置，返回空数组。

splice

向数组指定位置插入，删除，替换指定元素。

返回原始数组，在原数组上修改。

  • 删除：两个参数。arr.splice(删除位置下标, 删除的项数);
  • 插入：三个参数。arr.splice(起始位置, 0(删除项数), 插入项);
  • 替换：三个参数。arr.splice(起始位置, 删除项数, 插入项);向数组中先删除，在插入。

位置方法

ES5中出现。支持IE9+，FF2+，safari3+，Opera9.5，chrome

indexOf

arr.indexOf(要查找的项, (可选/默认0)查找起点的位置);

从数组指定位置开始向后查找，注意使用全等操作符比较。

未找到返回-1。

lastIndexOf

arr.lastIndexOf(要查找的项, (可选/默认末尾)查找起点的位置);

从数组指定位置开始向前查找，注意使用全等操作符比较。

未找到返回-1。

迭代方法

ES5增加5个方法，均接受两个参数：在每一项上运行的函数和（可选）运行该函数的作用域对象。

传入这些方法中的函数会接受三个参数：数组项的值，该项在数组中的位置，数组对象本身。

支持：IE9+，FF2，Safari3+，Opera9.5，chrome

every

在数组中的每一项运行给定函数，如果每一项都返回true，则返回true。如果有一项是false则直接返回false，不在运行。

filter

对数组中每一项运行给定函数，返回该函数会返回true项组成的数组。

forEach

对数组的每一项运行给给定函数。没有返回值。

map

对数组中每一行运行给定函数，返回每次函数调用的结果组成的数组。

some

对数组中每一项运行指定函数，如果该数组任一项返回true，则返回true。当有一项返回true时，直接返回true，下面的不在进行计算。

缩小方法

ES5增加reduce和reduceRight。迭代数组所有项，构建一个最终返回值。

接收参数两个：一个在每一项上调用的函数和（可选）作为缩小基数基础 的初始值。

调用函数接受四个参数：前一个值，当前值，项的索引，数组对象。这个函数返回的任何值都会作为第一个参数自动传递给下一项。注意：第一次迭代发生在数组第二项上，第一个参数是数组第一项，第二个参数是数组第二项。

支持：IE9+，FF3+，Safari4+，Opera10.5，chrome

reduce

从数组第一项开始，逐个遍历到最后。

reduceRight

从数组的最后一项开始，向前遍历到第一项。

方法总结

  • 堆栈方法：push，pop，shift，unshift
  • 操作方法：concat，slice，splice，join
  • 位置方法：indexOf，lastIndexOf
  • 迭代方法：every，some，filter，map，forEach
  • 缩小方法：reduce，reduceRight

注意：

  • reduce，reduceRight适用于查找数组最大／小值，累加等
  • map适用于对数组每一项执行相同的操作，例如求取每一项的平方。
  • forEach适用于利用数组执行循环操作。
  • some适用于判断数组是否含有某一项
  • every适用于判断数组是否都满足条件

javascript中的队列和栈

发表于 2018-06-23 | 分类于 javaScript |

队列

队列是一种特殊的线性结构，只允许在队列的首部进行删除操作（出队），而在队列的尾部进行插入操作（入队）。符合先进先出（First In First Out，FIFO）原则。

在javascript中，使用array.push()和array.shift()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

  • 小游戏：揭秘数字

规则:首先将第 1 个数删除，紧接着将第 2 个数放到 这串数的 尾，再将第 3 个数删除并将第 4 个数放到这串数的 尾，再将第 5 个数删除…… 直到剩下最后一个数，将最后一个数也删除。按照刚才删除的顺序，把这些删除的数连在一起就是需要的数字。数组：“6 3 1 7 5 8 9 2 4”

栈

栈是一种后进后出（Last In First Out，LIFO）的数据结构。栈限定了只能在一端（栈的顶部）进行插入和删除的操作。

在javascript中，使用array.push()和array.pop()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.pop()：从数组中移除末项并返回移除项。

  • 小游戏：验证回文
  • 小游戏：验证括号匹配

思路： 利用栈的后进后出规则，但遇到左括号末端添加，遇到匹配右括号末端推出。

  • 纸牌游戏

规则：将一副扑克牌平均分成两份，每人拿一份。一人先拿出手中的 第一张扑克牌放在桌上，然后另一人也拿出手中的第一张扑克牌，并放在刚打出的扑克牌 的上面，就像这样两人交替出牌。出牌时，如果某人打出的牌与桌上某张牌的牌面相同，即可将两张相同的牌及其中间所夹的牌全部取走，并依次放到自己手中牌的 尾。当任意一人手中的牌全部出完时，游戏结束，获胜。

javascript中插入删除数组总结：

array.pop()：从数组中移除末项并返回移除项。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

array.unshift()：向数组的第一项添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

javascript中的正则表达式

发表于 2018-06-22 | 分类于 javaScript |

ECMAScript通过RegExp数据类型来支持正则表达式。

字面量形式创建

var expression = / pattern / flags ;

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = /.at/gi;

  • pattern：

    任何简单或复杂的正则表达式。可包含字符类，限定符，分组，向前查找以及反向引用。

  • flags：标明正则表达式的行为，支持三个标志。
    • g：表示全局模式，即模式将被应用于所有字符串，而非发现第一个匹配时立即停止。
    • i：表示不区分大小写模式。
    • m：表示多行模式，即在一行文本末尾还会继续查找下一行是否存在与模式匹配的项。

使用RegExp构造函数创建

接收两个参数，第一个要匹配的字符串模式，第二个标志字符串。

var expression = new RegExp( "pattern", "flags" );

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = new RegExp(".at", "gi");

注意：双转义

使用构造函数传入必须为字符串，在某些情况下要对字符进行双重转义。

下面给出部分例子：

字面量模式 等价的字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/age/ "name\\\/age"
/\d.\d{1,2}/ "\\d.\\d{1,2}"
/\w\\hello\\123/ "\\w\\\\hello\\\\123"

在ES3中，正则表达式字面量始终为共享同一个RegExp实例，而使用构造函数创建的每一个新RegExp都是一个新的实例。

在ES5中，明确规定字面量模式也必须创建新的实例。IE9，Firefox4+，Chrome做出了修改。

常用正则表达基本概念

pattern可包含内容：

  • 普通字符：没有显示指定为元字符的所有可打印不可打印字符。

    • 非打印字符，需要进行转义。

  • 特殊字符

    具有特殊含义的字符，包括元字符。需要进行转义。

    使用\进行转义，例如我们需要匹配a?b则应该写成var exp = /a\?b/g;

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | \d | 匹配一个数字字符，等价于[0-9] |
    | \D | 匹配一个非数字字符，等价于[^0-9] |
    | \s | 匹配任何空白字符，包括空格，制表符，换页符等。等价于[\f\n\r\t\v] |
    | \S | 匹配任何非空白字符。等价于[^\f\n\r\t\v] |
    | \w | 匹配字母，数字，下划线。等价于[A-Za-z0-9_] |
    | \w | 匹配非字母，数字，下划线。等价于[^A-Za-z0-9_] |

  • 限定符

    指定正则表达式中一个给定组件出现的次数。

    | 字符 | 描述 |
    | —– | —————– |
    | * | 0或多次 |
    | + | 1或多次 |
    | ？ | 0或1 |
    | {n} | n次 |
    | {n,} | 大于等于n次 |
    | {n,m} | n-m次（包含n，m） |

    贪心模式和非贪心模式转换

    *,+都是贪心模式，尽可能匹配多的字符串，当在*,+后面放置?可以将贪心模式转换为非贪心模式（最小匹配）。

  • 定位符

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | ^ | 匹配输入字符串开始的位置。如果设置了m模式，还会与\n,\r之后的位置匹配。 |
    | $ | 匹配输入字符串结束的位置。如果设置了m模式，还会与\n,\r之前的位置匹配。 |
    | \b | 匹配一个字边界，即字与空格间的位置。 |
    | \B | 非字边界匹配。 |

  • 选择

    (选择1 | 选择2 | 选择3)

    圆括号副作用会使相关匹配被缓存，可使用?:消除缓存?:(选择1 | 选择2 | 选择3)

运算符优先级

从左到右进行计算，训训有限即规则

RegExp实例属性

RegExp每个实例都具有以下属性，可以获得有关模式信息

  • global：布尔值，是否设置g
  • ignoreCase：布尔值，是否设置i
  • lastIndex：整数，表示开始搜索下一个匹配项的字符位置，从0起算
  • multiline：布尔值，是否设置m
  • source：正则表达式的字符串表示，按照字面量形式字符串返回（即使使用构造函数创建）。
