数组乱序／洗牌问题

发表于 2018-06-26 | 分类于 algorithm |

将一个数组中的顺序打乱有多种方法，但是哪一种最高效呢（我也不知道。。。只能记录一下我会的这些方法哪一种更好）～

推荐使用方法三

题目：给定一个数组，将其打乱，返回新的数组，即为数组乱序，也称为洗牌问题

具条件，一是正确性，毋庸置疑，这是必须的，二是高效性，在确保正确的前提下，如何将复杂度降到最小。

方法一

思路： 每次随机random生成一个数组的下标，取出该项。剩余数组继续随机生成下标，直到全部取出。

缺点： 当数组非常大的时候会很慢。

整体复杂度：O(n^2).

该方法中使用splice，splice的复杂度是O(n)

方法二

思路：巧妙利用Math.random。这种方法仅仅适用于非正式场合

平均复杂度：O(nlogn)

方法三： Fisher–Yates shuffle

思路：遍历数组元素，将其与之前的任意元素交换。因为遍历有从前向后和从后往前两种方式，所以该算法大致也有两个版本的实现。

复杂度：O(n)

  • 从后向前
  • 从前向后

Number,String,Boolean

发表于 2018-06-25 | 分类于 javaScript |

当操作string，number，boolean基本类型的数据，按照基本类型的相应行为，它们不具有方法。但是它可以具有类似引用类型的特殊行为。原因在于当读取一个基本类型是，后台会常见一个对应的基本包装类型对象，从而使其具有方法。

当使用基本类型（boolean，number，sring）的特殊行为时，后台进行如下处理：

  1. 创建相应基本类型的实例
  2. 在实例上指定方法
  3. 销毁这个实例

与引用类型区别：

  • 对象生存期不同。

    引用类型实例在真个作用域销毁之前一直存在在内存中。

    自动创建基本包装类型对象，只存在于代码执行瞬间，然后立即被销毁。

    （因此项字符串添加属性不会报错，但是也不会被保存）

显示创建基本包装类型的对象

不推荐

可以使用new显示的创建String，Boolean，Number的对象，但是不推荐，因为使用这种方式创建的类型属于object。

  1. reverse 反转数组

  2. sort排序

    因此，sort可以接受一个比较函数作为参数。

    • 第一个参位于第二个参数之前，返回负数
    • 第一个参和第二个参数相等，返回0
    • 第一个参位于第二个参数之后，返回正数

操作方法

concat

基于当前数组创建一个副本，不接收或接收多个参数，并把传入的参数依次添加到副本的尾部。

返回当前数组的副本，不改变数组本身。

slice

接收一个或多个参数，指定返回项的开始和结束位置（不包括结束位置）。只有一个参数情况下，返回从指定位置到末尾所有项。

不改变数组本身。

注意：

  • 参数重有负数，则利用数组长度转换为正数。

    例如：length=5，slice(-2,1) = slice(3,4)

  • 结束位置小于起始位置，返回空数组。

splice

向数组指定位置插入，删除，替换指定元素。

返回原始数组，在原数组上修改。

  • 删除：两个参数。arr.splice(删除位置下标, 删除的项数);
  • 插入：三个参数。arr.splice(起始位置, 0(删除项数), 插入项);
  • 替换：三个参数。arr.splice(起始位置, 删除项数, 插入项);向数组中先删除，在插入。

位置方法

ES5中出现。支持IE9+，FF2+，safari3+，Opera9.5，chrome

indexOf

arr.indexOf(要查找的项, (可选/默认0)查找起点的位置);

从数组指定位置开始向后查找，注意使用全等操作符比较。

未找到返回-1。

lastIndexOf

arr.lastIndexOf(要查找的项, (可选/默认末尾)查找起点的位置);

从数组指定位置开始向前查找，注意使用全等操作符比较。

未找到返回-1。

迭代方法

ES5增加5个方法，均接受两个参数：在每一项上运行的函数和（可选）运行该函数的作用域对象。

传入这些方法中的函数会接受三个参数：数组项的值，该项在数组中的位置，数组对象本身。

支持：IE9+，FF2，Safari3+，Opera9.5，chrome

every

在数组中的每一项运行给定函数，如果每一项都返回true，则返回true。如果有一项是false则直接返回false，不在运行。

filter

对数组中每一项运行给定函数，返回该函数会返回true项组成的数组。

forEach

对数组的每一项运行给给定函数。没有返回值。

map

对数组中每一行运行给定函数，返回每次函数调用的结果组成的数组。

some

对数组中每一项运行指定函数，如果该数组任一项返回true，则返回true。当有一项返回true时，直接返回true，下面的不在进行计算。

缩小方法

ES5增加reduce和reduceRight。迭代数组所有项，构建一个最终返回值。

接收参数两个：一个在每一项上调用的函数和（可选）作为缩小基数基础 的初始值。

调用函数接受四个参数：前一个值，当前值，项的索引，数组对象。这个函数返回的任何值都会作为第一个参数自动传递给下一项。注意：第一次迭代发生在数组第二项上，第一个参数是数组第一项，第二个参数是数组第二项。

支持：IE9+，FF3+，Safari4+，Opera10.5，chrome

reduce

从数组第一项开始，逐个遍历到最后。

reduceRight

从数组的最后一项开始，向前遍历到第一项。

方法总结

  • 堆栈方法：push，pop，shift，unshift
  • 操作方法：concat，slice，splice，join
  • 位置方法：indexOf，lastIndexOf
  • 迭代方法：every，some，filter，map，forEach
  • 缩小方法：reduce，reduceRight

注意：

  • reduce，reduceRight适用于查找数组最大／小值，累加等
  • map适用于对数组每一项执行相同的操作，例如求取每一项的平方。
  • forEach适用于利用数组执行循环操作。
  • some适用于判断数组是否含有某一项
  • every适用于判断数组是否都满足条件

javascript中的队列和栈

发表于 2018-06-23 | 分类于 javaScript |

队列

队列是一种特殊的线性结构，只允许在队列的首部进行删除操作（出队），而在队列的尾部进行插入操作（入队）。符合先进先出（First In First Out，FIFO）原则。

在javascript中，使用array.push()和array.shift()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

  • 小游戏：揭秘数字

规则:首先将第 1 个数删除，紧接着将第 2 个数放到 这串数的 尾，再将第 3 个数删除并将第 4 个数放到这串数的 尾，再将第 5 个数删除…… 直到剩下最后一个数，将最后一个数也删除。按照刚才删除的顺序，把这些删除的数连在一起就是需要的数字。数组：“6 3 1 7 5 8 9 2 4”

栈

栈是一种后进后出（Last In First Out，LIFO）的数据结构。栈限定了只能在一端（栈的顶部）进行插入和删除的操作。

在javascript中，使用array.push()和array.pop()可以模仿队列的入队和出队。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.pop()：从数组中移除末项并返回移除项。

  • 小游戏：验证回文
  • 小游戏：验证括号匹配

思路： 利用栈的后进后出规则，但遇到左括号末端添加，遇到匹配右括号末端推出。

  • 纸牌游戏

规则：将一副扑克牌平均分成两份，每人拿一份。一人先拿出手中的 第一张扑克牌放在桌上，然后另一人也拿出手中的第一张扑克牌，并放在刚打出的扑克牌 的上面，就像这样两人交替出牌。出牌时，如果某人打出的牌与桌上某张牌的牌面相同，即可将两张相同的牌及其中间所夹的牌全部取走，并依次放到自己手中牌的 尾。当任意一人手中的牌全部出完时，游戏结束，获胜。

javascript中插入删除数组总结：

array.pop()：从数组中移除末项并返回移除项。

array.push()：向数组的末端添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

array.shift()：从数组中移除第一项并返回移除项。

array.unshift()：向数组的第一项添加项，返回数组添加后的长度。可以同时添加多项，用逗号隔开。

javascript中的正则表达式

发表于 2018-06-22 | 分类于 javaScript |

ECMAScript通过RegExp数据类型来支持正则表达式。

字面量形式创建

var expression = / pattern / flags ;

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = /.at/gi;

  • pattern：

    任何简单或复杂的正则表达式。可包含字符类，限定符，分组，向前查找以及反向引用。

  • flags：标明正则表达式的行为，支持三个标志。
    • g：表示全局模式，即模式将被应用于所有字符串，而非发现第一个匹配时立即停止。
    • i：表示不区分大小写模式。
    • m：表示多行模式，即在一行文本末尾还会继续查找下一行是否存在与模式匹配的项。

使用RegExp构造函数创建

接收两个参数，第一个要匹配的字符串模式，第二个标志字符串。

var expression = new RegExp( "pattern", "flags" );

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = new RegExp(".at", "gi");

注意：双转义

使用构造函数传入必须为字符串，在某些情况下要对字符进行双重转义。

下面给出部分例子：

字面量模式 等价的字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/age/ "name\\\/age"
/\d.\d{1,2}/ "\\d.\\d{1,2}"
/\w\\hello\\123/ "\\w\\\\hello\\\\123"

在ES3中，正则表达式字面量始终为共享同一个RegExp实例，而使用构造函数创建的每一个新RegExp都是一个新的实例。

在ES5中，明确规定字面量模式也必须创建新的实例。IE9，Firefox4+，Chrome做出了修改。

常用正则表达基本概念

pattern可包含内容：

  • 普通字符：没有显示指定为元字符的所有可打印不可打印字符。

    • 非打印字符，需要进行转义。

  • 特殊字符

    具有特殊含义的字符，包括元字符。需要进行转义。

    使用\进行转义，例如我们需要匹配a?b则应该写成var exp = /a\?b/g;

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | \d | 匹配一个数字字符，等价于[0-9] |
    | \D | 匹配一个非数字字符，等价于[^0-9] |
    | \s | 匹配任何空白字符，包括空格，制表符，换页符等。等价于[\f\n\r\t\v] |
    | \S | 匹配任何非空白字符。等价于[^\f\n\r\t\v] |

这次主要记录一下冒泡排序和快速排序～

冒泡排序

  • 基本思想：

    每次比较两个相邻的元素，如果它们的顺序错误就把它们交换过来。

  • 总结:

    如果有 n 个数进行排序，只需将 n-1 个数归位，也就是说要进行n-1 趟操作。而“每一趟”都需要从第 1 位开始进行相邻两个数的比较，将较小的一个数放在后面，比较完毕后向后挪一位继续比较下面两个相邻数的大小，重复此步骤，直到最后一个尚未归位的数，已经归位的数则无需再进行比较。

    O(N^2)

快速排序

  • 原理：

    为了方便记录以数组arr = [6, 1, 2, 7, 9, 3, 4, 5, 10, 8]为例。

    1. 首先选取一个基数。这里选取第一个数值。

      base=arr[0]=6;

    2. 从数组最右边开始（j = arr.length）向左移动寻找小于基数的数值。

      j=9开始，在j=7， arr[7]=5停止。

    3. 从数组最左边开始（j=0）向最右边移动寻找大于基数的数值。

      i=0开始，在i=3， arr[3]=7停止。

    4. 当i,j不相等时，交换i,j所对应的数值。重复2，3步骤。

      [6, 1, 2, 5, 9, 3, 4, 7, 10, 8]

      重复2，3步骤，继续查找。在j=6，arr[6]=4，i=4，arr[i]=9停止。

      [6, 1, 2, 5, 4 , 3, 9 , 7, 10, 8]

    5. 当ij相等时，交换基数和i所对应数值。

      继续步骤2，当j=5时，j停止

      继续步骤3，当i=5时，i==j

      当i==j时，交换base和i的数值。

      [3, 1, 2, 5, 4 , 6, 9 , 7, 10, 8]

    6. 交换基数后，在基数左边的数全部小于基数，基数右边的数全部大于基数。完成了第一次排序。接下来，需要对基数左边和基数右边的书分别重复上面操作，依次迭代。

      左边[3,1,2,5,4]，右边[9,7,18,8]分别执行步骤1-5。直到基数左边和右边仅有一个数值，不在需要进行排序。

    O (NlogN) 平均时间复杂度

BFC

发表于 2018-06-12 | 分类于 css |

BFC 即Box Formatting Context的缩写，是一种css样式渲染的方式，可以简单理解为在BFC环境内的元素将按照BFC方式进行渲染。

最常见的 Formatting context 有 Block fomatting context (简称BFC)和 Inline formatting context (简称IFC)。CSS3 中还增加了 GFC和 FFC。

BFC(Block formatting context)直译为”块级格式化上下文”。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

BFC布局规则：

  1. 内部的块元素会换行排列。
  2. 属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠。
  3. 每个元素默认与父容器border-left对齐，即使存在浮动也是如此。
  4. BFC的区域不会与float元素区域重叠。（注意： 此处float元素不包含在这个不重叠BFC中，即不是子元素）
  5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  6. 计算BFC的高度时，浮动元素也参与计算。（注意：默认（不是BFC）不会计算，会导致盒子塌陷）

BFC触发条件：

  1. 根元素
  2. float属性不为none
  3. position为absolute或fixed
  4. display为inline-block, table-cell, table-caption, flex, inline-flex
  5. overflow不为visible

应用：

  1. 浮动盒子塌陷问题（清除浮动）

    应用BFC布局原则6：计算BFC的高度时，浮动元素也参与计算。

    ​

  1. float文字环绕问题

    应用BFC布局原则4：BFC的区域不会与float元素区域重叠。。

  2. 防止垂直 margin 重叠

    应用BFC布局原则2／5：属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠；BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。

基础概念注意问题

XHTML与HTML区别

  • XHTML标签名必须小写，标签必须严格嵌套，标签（包括空元素）必须封闭。
  • XHTML属性名必须小写，属性值必须用双引号括起来，属性值必须使用完整形式。
  • XHTML区分内容标签和结构标签。

在HTML中引入CSS的方式

方式分类

  • 行内式：在标签的style属性中设置样式

  • 嵌入式：在heade中style标签內添加样式

  • 导入式和链接式：将独立CSS文件引入到HTML中

    • 链接式：

      <link rel="stylesheet" href="style.css">

    • 导入式：

区别

  • 行内式：优先级高于其它方式，但不利于复用和维护，不推荐使用。
  • 嵌入式：针对单一页面方便，多页面网站复用性差。
  • 链接式：在装载页面主体之前装载CSS样式，页面加载出来就带有样式。引入多个CSS文件，调整CSS分类需同时调整HTML文件。
  • 导入式：在整个页面装载完成后装载CSS样式，网页文件体积较大时会出现抖动。注意使用导入式应将@import语句写在style的开始位置，即其它样式之前。
  • 建议：当引入多个CSS文件时，可以使用链接式引入一个总CSS文件，在总CSS文件中使用导入式引入其它CSS文件。

  • 允许任意列的高度最高；

圣杯布局

经典三列布局，也叫做圣杯布局【Holy Grail of Layouts】是Kevin Cornell在2006年提出的一个布局模型概念。

圣杯布局是一个相对定位，通过使用float，marginleft，relative的left／ringht布局。

  1. 设置基本样式

  2. 给.container添加padding预留出.left，.right的位置

  3. 给.left，.right，.main添加浮动

  4. 使用margin-left将.left，.right移动到与.main同行

    注意：

    margin-left

    • 负的margin-left会让元素沿文档流向左移动，如果负的数值比较大就会一直移动到上一行。
    • 设置left部分的margin-left为-100%,就会使left向左移动一整个行的宽度，由于left左边是父元素的边框，所以left继续跳到上一行左移，一直移动到上一行的开头，并覆盖了main部分，left上移过后，right就会处于上一行的开头位置，这时再设置right部分margin-left为负的宽度，right就会左移到上一行的末尾。

  5. 使用left,right将.left，.right移动到.container的padding部分
