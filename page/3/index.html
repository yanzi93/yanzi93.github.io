flex布局

发表于 2018-05-29 | 分类于 css |

flex即(Flexible Box)的缩写，意思为弹性盒模型。2009年由W3C提出的一种新的布局方案，目前已得到所有浏览器支持。

注意：

  • 任何元素都可以指定为flex布局，包括行内元素（inline-flex）。
  • 子元素float，clear，vertical-align属性失效。

flex基本概念

可以把flex元素想象成一个容器，容器中所有子元素自动成为容器成员（flex item）。

flex容器默认存在两根轴：主轴（main-axis，默认水平），垂直交叉轴（cross-axis，与主轴垂直。）

  • flex-start：主轴起始点（main start）或交叉轴起始点（cross start）
  • flex-end：主轴终点（main start）或交叉轴终点（cross end）
  • main size：每个容器成员（子元素）沿主轴的大小。
  • cross size： 每个容器成员（子元素）沿交叉轴的大小。

容器元素设置属性

  • diaplay: flex;
  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content

首先我们必须对容器元素设置display: flex; 设置弹性盒模型diaplay: flex;使其成为弹性盒模型。然后对其属性进行进一步设置。

flex-direction

设置主轴方向，即项目的排列方向。

  • row（默认值）：主轴为水平方向，起点在左端。
  • row-reverse：主轴为水平方向，起点在右端。
  • column：主轴为垂直方向，起点在上沿。
  • column-reverse：主轴为垂直方向，起点在下沿。

下图分别为column-reverse，column，row，row-reverse

flex-wrap

设置容器项目在沿主轴排列不下时，是否进行换行。默认不进行换行，当宽度超出时会进行挤压（挤压到内部元素撑开最小宽度），但border不会进行挤压。

  • nowrap（默认值）：不换行。当宽度超出时会进行挤压（挤压到内部元素撑开最小宽度），但border不会进行挤压。
  • wrap：当一行排列不下时，进行换行。第一行在上方。
  • wrap-reverse：当一行排列不下时，进行换行。第一行在下方。

下图分别为nowrap，wrap，wrap-reverse

flex-flow

flex-flow是flex-direction和flex-wrap的缩写。默认值时row nowrap。

justify-content

设置容器项目（flex item）沿主轴的对齐分布方式。

  • flex-start（默认值）：左对齐
  • flex-end：右对齐
  • center： 居中
  • space-between：两端对齐，项目之间的间隔都相等。
  • space-around：每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍。

align-items

设置容器项目（flex item）沿交叉轴的对齐分布方式。

  • flex-start：交叉轴的起点对齐。
  • flex-end：交叉轴的终点对齐。
  • center：交叉轴的中点对齐。
  • baseline: 项目的第一行文字的基线对齐。
  • stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。

下图分别为flex-start，flex-end，center，baseline，stretch。

align-content

属性定义了交叉线的对齐方式。如果项目只有一根轴线（即没有进行换行），该属性不起作用。

  • flex-start：与交叉轴的起点对齐。
  • flex-end：与交叉轴的终点对齐。
  • center：与交叉轴的中点对齐。
  • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
  • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
  • stretch（默认值）：轴线占满整个交叉轴。

容器项目（flex item）设置属性

  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex
  • align-self

对于每一个容器项目可以设置它的属性。

order

定义项目的排列顺序。数值越小，排列越靠前，默认为0。

可以设置每一个容器项目的order，不同order情况下，order越大排列越靠后。相同order情况下，按照容器项目本身顺序进行排序。

flex-grow

定义容器项目的放大比例，默认为0，即如果存在剩余空间，也不放大。

当设置flex-grow，且存在剩余空间时，该容器项目会进行放大，占用剩余空间。

当所有项目flex-grow相同时，平分剩余空间。当项目flex-grow不同时，按照分数进行分配。

flex-shrink

定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。

如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。负值无效。

flex-basis

定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。

与设置容器项目width／height相同，当超出时会被压缩。

flex

属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选。

该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。

建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值。

align-self

属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch

注意

  • 当容器项目设置绝对定位会使其脱离文档流，不受父元素控制，也会影响父元素子集其他元素分布。

绝对定位会使其不脱离文档流，但是位置可以进行进一步精确。

  • 当父元素宽高无法定死，很难预料其分布。

Vuex

简介

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

简单按照我的理解：vuex就是将vue不同组件中用到的相同数据进行统一管理，减少了父子组件，兄弟组件传值所引起的单项数据流不确定性和提高后期维护。

如果您不打算开发大型单页应用，使用 Vuex 可能是繁琐冗余的。确实是如此——如果您的应用够简单，您最好不要使用 Vuex。一个简单的 global event bus 就足够您所需了。但是，如果您需要构建一个中大型单页应用，您很可能会考虑如何更好地在组件外部管理状态，Vuex 将会成为自然而然的选择。

基本思想

State —dispatch—>Actions — commit — >Mutations — mutate— >State

按照上面进行循环。

state：存储共享数据

action： 当我们操作共享数据时，使用dispatch方法将数据进行传递

mutations：操作数据

每一个 Vuex 应用的核心就是 store（仓库）。“store”基本上就是一个容器，它包含着你的应用中大部分的状态 (state)。Vuex 和单纯的全局对象有以下两点不同：

  1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。
  2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。

排序方法（归并排序，冒泡排序，快速排序）

归并排序

归并排序是建立在归并操作上的一种有效的排序算法。该算法是采用分治法（Divide and Conquer）的一个非常典型的应用。

归并排序，首先归是讲数组以长度中点进行拆分，直到拆分到单个元素。然后进行并，也就是将元素排序合并。

快速排序

交换排序的一种。

快速排序就是在数组中选一个数为基数，比基数小的数放在左边，比基数大的数放在基数右边。在选择一个基数进行同样操作，直到排序完成。

冒泡排序

交换排序的一种

四种排序的比较

排序方法 时间复杂度 最好时间复杂度 最坏时间复杂度 空间复杂度 稳定性 复杂性
堆排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(nlog_2n)$ $O(1)$ 不稳定 较复杂
递归排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(nlog_2n)$ $O(n)$ 稳定 较复杂
冒泡排序 $O(n^2)$ $O(n)$ $O(n^2)$ $O(1)$ 稳定 简单
快速排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(n^2)$ $O(nlog_2n)$ 不稳定 较复杂

稳定性

假定在待排序的记录序列中，存在多个具有相同的关键字的记录，若经过排序，这些记录的相对次序保持不变，即在原序列中，ri=rj，且ri在rj之前，而在排序后的序列中，ri仍在rj之前，则称这种排序算法是稳定的；否则称为不稳定的。

技术简介

###HTML5 应用场景

  • 新增加API接口
    • Local storage: 提高用户本地存储。
    • Web Worker: 后台计算线程，所有与UI相关的不能使用；解决js单线程问题，能够把js计算委托给后台线程，防止交互事件缓慢；在API接口提供前使用setTimeout。
    • XMLHttpRequest： 解决跨域请求。
  • 基于webview 的hybrid native App开发更加灵活。

CSS3

  • 体积小。
  • 对于webApp近乎不用考虑兼容问题。

Ajax

  • 在不重新加载页面情况下，与服务器交换数据并更新部分网页内容
  • 不需要任何插件

ES6 Promise

  • 一个保存某个未来才会结束的事件
  • 特点：
    • 不受外间干扰。只有pending，fulfilled，rejected状态，由异步造作结果决定。
    • 状态只改变一次。
    • 相比回调函数，避免层层嵌套。
  • 缺点：
    • 一旦建立立即执行
    • 不但发生的事件，不建议使用

移动端性能问题

  • 减少或避免repaint，reflow。
    • 减少对DOM的操作
    • 改变Class而不是style，使用classList而不是className
    • 避免使用docuement.write
    • 减少drawImage
  • 缓存一切可以缓存的数据
    • 使用HTML5的local.stroage减少向服务器的请求次数。
  • 使用CSS3动画代替DOM操作
  • 适当使用硬件加速
    • CSS3 中transitions、CSS3 3D transforms、Opacity、Canvas、WebGL、Video会触发GPU加速

开发流程

交互解读 - 设计解读 - 开发 - 测试

技术原则： 轻量化，维护简单，快速开发，高性能
