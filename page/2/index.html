排序方法（归并排序，冒泡排序，快速排序）

归并排序

归并排序是建立在归并操作上的一种有效的排序算法。该算法是采用分治法（Divide and Conquer）的一个非常典型的应用。

归并排序，首先归是讲数组以长度中点进行拆分，直到拆分到单个元素。然后进行并，也就是将元素排序合并。

快速排序

交换排序的一种。

快速排序就是在数组中选一个数为基数，比基数小的数放在左边，比基数大的数放在基数右边。在选择一个基数进行同样操作，直到排序完成。

冒泡排序

交换排序的一种

四种排序的比较

排序方法 时间复杂度 最好时间复杂度 最坏时间复杂度 空间复杂度 稳定性 复杂性
堆排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(nlog_2n)$ $O(1)$ 不稳定 较复杂
递归排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(nlog_2n)$ $O(n)$ 稳定 较复杂
冒泡排序 $O(n^2)$ $O(n)$ $O(n^2)$ $O(1)$ 稳定 简单
快速排序 $O(nlog_2n)$ $O(nlog_2n)$ $O(n^2)$ $O(nlog_2n)$ 不稳定 较复杂

稳定性

假定在待排序的记录序列中，存在多个具有相同的关键字的记录，若经过排序，这些记录的相对次序保持不变，即在原序列中，ri=rj，且ri在rj之前，而在排序后的序列中，ri仍在rj之前，则称这种排序算法是稳定的；否则称为不稳定的。

webApp 书城开发(一阅读器页面部分)

小谈

记录慕课网webApp 书城开发实战课程

涉及技术

HTML5，CSS3，javascript，Ajax，zepto，vue，ES6，nodejs，npm，KOA

技术简介

###HTML5 应用场景

  • 新增加API接口
    • Local storage: 提高用户本地存储。
    • Web Worker: 后台计算线程，所有与UI相关的不能使用；解决js单线程问题，能够把js计算委托给后台线程，防止交互事件缓慢；在API接口提供前使用setTimeout。
    • XMLHttpRequest： 解决跨域请求。
  • 基于webview 的hybrid native App开发更加灵活。

CSS3

  • 体积小。
  • 对于webApp近乎不用考虑兼容问题。

Ajax

  • 在不重新加载页面情况下，与服务器交换数据并更新部分网页内容
  • 不需要任何插件

ES6 Promise

  • 一个保存某个未来才会结束的事件
  • 特点：
    • 不受外间干扰。只有pending，fulfilled，rejected状态，由异步造作结果决定。
    • 状态只改变一次。
    • 相比回调函数，避免层层嵌套。
  • 缺点：
    • 一旦建立立即执行
    • 不但发生的事件，不建议使用

移动端性能问题

  • 减少或避免repaint，reflow。
    • 减少对DOM的操作
    • 改变Class而不是style，使用classList而不是className
    • 避免使用docuement.write
    • 减少drawImage
  • 缓存一切可以缓存的数据
    • 使用HTML5的local.stroage减少向服务器的请求次数。
  • 使用CSS3动画代替DOM操作
  • 适当使用硬件加速
    • CSS3 中transitions、CSS3 3D transforms、Opacity、Canvas、WebGL、Video会触发GPU加速

开发流程

交互解读 - 设计解读 - 开发 - 测试

技术原则： 轻量化，维护简单，快速开发，高性能
