git

git命令

  1. git init
  2. git add . 添加到本地缓存中
  3. git commit -m "status"
  4. git push 提交到仓库中
  5. git pull 将仓库分支加载到本地
  6. git checkout branch-name 切换分支
  7. git status 查看状态
  8. git merge origin/index-swiper 将分枝index-swiper的内容合并到主分支上，合并后使用git push 进行上传，一般我们会在分支进行编写到可直接使用稳定版本后进行合并。
  9. npm run start 启动

开发时git分支切换步骤

在进行开发时，我们通常在分支上进行组件的开发，当开发结束后，将最后版本添加到主分支上进行合并。

  1. 在git仓库中创建新的分支。
  2. 在本地中使用git pull 将分支加载到 本地。
  3. 本地使用git check branch-name 切换到分支上进行开发。
  4. 可以使用git status 查看是否切换成功。
  5. 使用git add . git commit -m "status" git push 上传到仓库分支上。

分支内容合并到主分支

当分支内容开发完毕后会将分支内容合并到主分支上。

  1. git checkout master 切换到主分支上
  2. git merge origin/branch-name branch-name为需要合并分支的名字，对分支内容进行合并
  3. git push 提交到主分支仓库

Vuex

简介

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

简单按照我的理解：vuex就是将vue不同组件中用到的相同数据进行统一管理，减少了父子组件，兄弟组件传值所引起的单项数据流不确定性和提高后期维护。

如果您不打算开发大型单页应用，使用 Vuex 可能是繁琐冗余的。确实是如此——如果您的应用够简单，您最好不要使用 Vuex。一个简单的 global event bus 就足够您所需了。但是，如果您需要构建一个中大型单页应用，您很可能会考虑如何更好地在组件外部管理状态，Vuex 将会成为自然而然的选择。

基本思想

State —dispatch—>Actions — commit — >Mutations — mutate— >State

按照上面进行循环。

state：存储共享数据

action： 当我们操作共享数据时，使用dispatch方法将数据进行传递

mutations：操作数据

每一个 Vuex 应用的核心就是 store（仓库）。“store”基本上就是一个容器，它包含着你的应用中大部分的状态 (state)。Vuex 和单纯的全局对象有以下两点不同：

  1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。
  2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。

Keep-alive

使用keep-alive包裹的模版，在一次加载后放入内存，防止进行多次加载降低性能。

使用：

额外增加一个activated生命钩子函数，首次进入页面mounted，activated都会被执行，但当页面重新显示时activated就会被执行，但是因为使用keep-alive mounted函数不会被执行。

同时增加一个deactivated生命钩子函数，在页面即将更换成新的页面或关闭时执行。我们一般在这个函数內对全局事件进行解绑。

可以使用<keep-alive exclude="Detail">排除不需要缓存的组件。
