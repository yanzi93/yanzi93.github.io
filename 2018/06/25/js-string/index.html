Number,String,Boolean

发表于 2018-06-25 | 分类于 javaScript |

当操作string，number，boolean基本类型的数据，按照基本类型的相应行为，它们不具有方法。但是它可以具有类似引用类型的特殊行为。原因在于当读取一个基本类型是，后台会常见一个对应的基本包装类型对象，从而使其具有方法。

当使用基本类型（boolean，number，sring）的特殊行为时，后台进行如下处理：

  1. 创建相应基本类型的实例
  2. 在实例上指定方法
  3. 销毁这个实例

与引用类型区别：

  • 对象生存期不同。

    引用类型实例在真个作用域销毁之前一直存在在内存中。

    自动创建基本包装类型对象，只存在于代码执行瞬间，然后立即被销毁。

    （因此项字符串添加属性不会报错，但是也不会被保存）

显示创建基本包装类型的对象

不推荐

可以使用new显示的创建String，Boolean，Number的对象，但是不推荐，因为使用这种方式创建的类型属于object。

Boolean类型

不推荐使用

首先我们要区分Boolean类型和Boolean基本类型。

Boolean基本类型取值包括：true，false

Boolean引用类型是使用new构建的。调用Boolean构造函数，传入true，false。

Number类型

不推荐

使用new创建Number引用类型

方法

具有的方法使用Number类型同样适用。

toFixed

按照指定的小数返回数值的字符串表示。

接收一个参数：输出结果中小数位数

不同浏览器舍入规则不同。IE8及之前不能正确处理{(-0.94, -0.5 ], [0.5, 0.94)}之间数值。

一般支持0-20位小数

toExponential

返回以指数表示法表示的数值的字符串形式。

接收一个参数：输出结果中小数位数

toPrecision

返回适合格式的指定小数位数的字符串。

接收一个参数：输出结果中小数的位数，可能按照toFixed返回也可能按照toExponential返回。

String类型

使用String类型的每个方法都可以直接使用String基本类型使用。

属性

length

返回字符串长度。激励含有双字节每个字符仍然算一个字符。

方法

字符方法

charAt

参数：接收一个基于零的字符位置

返回：以单字符字符串的形式返回给定位置的字符

charCodeAt

参数：接收一个基于零的字符位置

返回：以单字符字符串的形式返回给定位置的字符编码

直接使用下标索引

ES5新增，IE7及之前返回undefined

字符串操作方法

concat

用于奖一个或多个字符串拼接起来，返回拼接得到的新字符串。实际中更多实用+号进行拼接。

slice,substring,substr

参数

方法 参数1 参数2（缺省：字符串长度作为结束位置）
slice 子字符串开始位置（包含） 子字符串结束位置（不包含）
substring 子字符串开始位置（包含） 子字符串结束位置（不包含）
substr 子字符串开始位置（包含） 子字符串长度（包含）

第一个参数为负数

方法 参数1为负数
slice length+(argument1)
substring 全部字符串
substr length+(argument1)

第二个参数为负数

方法 参数2为负数
slice length+(argument1)
substring 0（substring会自动将较小的数字当作起始值）
substr 0（返回空字符串）

字符串位置方法

indexOf,lastIndexOf

从字符串中搜索给定的子字符串，返回子字符串的位置（未找到返回-1）。

参数：查找字符串，查找起始位置（可选）

应用：查找所有匹配的子字符串

trim

创建字符串的副本，删除前置及后缀的所有空格，饭后返回结果。

IE9+,FF3.5+,Safari5+,Opera10.5,chrome支持

FF3.5+，safari5+，chrome8+支持非标准trimLeft，teimRight

字符串大小写转换

toLowerCase，toUpperCase

字符串模式匹配方法

match

参数：接收一个参数，正则表达式或者字符串

输出：返回一个数组。当参数没有设置全局搜索g时与RegExp对象exec相同。当设置g时，返回左右匹配字符串组成的数组。

  • 当不设置全局搜索g时，返回第一个匹配的项目就停止，再次执行还是相同结果。

    返回的是对象，0:匹配项全部，1-n:匹配组，index：匹配项起始下标，input：输入数组

  • 当设置全局搜索时，match返回所有匹配项目组成的数组，exec返回匹配的第一项，再次执行在此基础上搜索下一项。

search

参数：接收一个参数，正则表达式或者字符串

输出：返回第一个匹配项的索引。未找到返回-1

始终从字符串开头向后查找

replace

参数：两个，第一个参数需要匹配的regexp或字符串（不会被转换成正则表达式），第二个参数时字符串或者函数

注意：第一个参数设置为全局g正则表达式才可以替换全部字符串。

  • 当第二个参数为字符串时，可以使用它数字符序列将正则表达式得到的值插入到结果字符串中。

    | 字符序列 | 替换文本 |
    | ——– | ———————————————————— |
    | $$ | $ |
    | $& | 匹配整个模式的字符串 |
    | $’ | 匹配子字符串之前的子字符串 |
    | $` | 匹配子字符串之后的子字符串 |
    | $n | 匹配第n个捕获组的子字符串，其中n等于0-9。如果正则表达式中没有定义捕获组，则使用空字符串。 |
    | $nn | 匹配第nn个捕获组的字符串，其中nn等于01-99。如果正则表达式中没有定义捕获组，则使用空字符串。 |

  • 当第二个参数为函数时，可以对匹配项进行更精密的操作

split

基于指定的分隔符将一个字符串分割成多个字符串组成的数组。

参数：两个参数，第一个参数字符串或regexp对象，第二个参数（可选）指定数组的大小，确保返回的数组不会超过既定大小。

注意：简单匹配浏览器无差别，对于包含捕获组或者未发现匹配项的模式匹配行为不同。这里展示chrome结果

localeCompare

比较两个字符串，并按下面规则返回

  • 字符串在字母表中排在字符串参数之前，返回负数
  • 字符串在字母表中排在字符串参数之后，返回正数
  • 相等，返回0

支持地区决定方法行为，不同地区排序可能不同。

fromCharCode

将字符编码转换为字符串，与charCodeAt行为相反。
