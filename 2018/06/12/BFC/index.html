BFC

发表于 2018-06-12 | 分类于 css |

BFC 即Box Formatting Context的缩写，是一种css样式渲染的方式，可以简单理解为在BFC环境内的元素将按照BFC方式进行渲染。

最常见的 Formatting context 有 Block fomatting context (简称BFC)和 Inline formatting context (简称IFC)。CSS3 中还增加了 GFC和 FFC。

BFC(Block formatting context)直译为”块级格式化上下文”。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

BFC布局规则：

  1. 内部的块元素会换行排列。
  2. 属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠。
  3. 每个元素默认与父容器border-left对齐，即使存在浮动也是如此。
  4. BFC的区域不会与float元素区域重叠。（注意： 此处float元素不包含在这个不重叠BFC中，即不是子元素）
  5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  6. 计算BFC的高度时，浮动元素也参与计算。（注意：默认（不是BFC）不会计算，会导致盒子塌陷）

BFC触发条件：

  1. 根元素
  2. float属性不为none
  3. position为absolute或fixed
  4. display为inline-block, table-cell, table-caption, flex, inline-flex
  5. overflow不为visible

应用：

  1. 浮动盒子塌陷问题（清除浮动）

    应用BFC布局原则6：计算BFC的高度时，浮动元素也参与计算。

    ​

  1. float文字环绕问题

    应用BFC布局原则4：BFC的区域不会与float元素区域重叠。。

  2. 防止垂直 margin 重叠

    应用BFC布局原则2／5：属于同一个BFC的两个相邻块元素的margin-top,margin-bottom会发生重叠；BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
