基础概念注意问题

XHTML与HTML区别

  • XHTML标签名必须小写，标签必须严格嵌套，标签（包括空元素）必须封闭。
  • XHTML属性名必须小写，属性值必须用双引号括起来，属性值必须使用完整形式。
  • XHTML区分内容标签和结构标签。

在HTML中引入CSS的方式

方式分类

  • 行内式：在标签的style属性中设置样式

  • 嵌入式：在heade中style标签內添加样式

  • 导入式和链接式：将独立CSS文件引入到HTML中

    • 链接式：

      <link rel="stylesheet" href="style.css">

    • 导入式：

区别

  • 行内式：优先级高于其它方式，但不利于复用和维护，不推荐使用。
  • 嵌入式：针对单一页面方便，多页面网站复用性差。
  • 链接式：在装载页面主体之前装载CSS样式，页面加载出来就带有样式。引入多个CSS文件，调整CSS分类需同时调整HTML文件。
  • 导入式：在整个页面装载完成后装载CSS样式，网页文件体积较大时会出现抖动。注意使用导入式应将@import语句写在style的开始位置，即其它样式之前。
  • 建议：当引入多个CSS文件时，可以使用链接式引入一个总CSS文件，在总CSS文件中使用导入式引入其它CSS文件。

替换元素

除了经常区分的块元素（block），內联块元素（inline-block），內联元素（inline）。

还注意区分替换元素（replace element）与非替换元素（noreplace element）

  • replace element：

    替换元素指用来替换元素内容的部分并非由文档内容直接表示。

    例如：img,input

  • noreplace element：

    大多数标签都是非替换元素，其内容由用户代理在元素本身生成的框中显示。

CSS选择器

CSS选择器的作用就是为了让CSS样式与HTML中标签对应起来。

  • 标签选择器 p {}
  • 类选择器 .class {}
  • id选择器 #id {}
    • id必须是唯一的，虽然不懂标签使用相同id会使其样式相同，但是在js中document.getElementById()只会获得第一个id的标签。
    • id必须只能有一个，不可以像class一样连写。
  • 交集选择器 h3.class{}或h3#id{}具有class类的标签或具有id的标签，两者连写不能中间有空格。
  • 并集选择器h3, .class{}h3和具有.class类的标签，两者中间使用逗号隔开。
  • 后代选择器.class h3{}具有.class类标签下面的h3标签，两者使用空格连接。所有子代包括非直接后代。
  • 子选择器.class{}>h3具有.class类标签下面的直接子代h3标签，两者使用空格连接。只包括直接子代，不包括孙子代等，IE6不支持。
  • 相邻兄弟选择器div+span{}与div标签相邻的span的标签，div与span是兄弟关系。
  • 属性选择器input[type="radio"]{},div[classz~="box"]{} = div .box{}IE6不支持。
  • 伪类选择器（pseudo-class selector）
    • :link,:visted,:hover,:active
    • :focus
    • :firsr-child
    • :first-letter,:first-line
    • :before,:after

CSS继承性（inheritance）与层叠性

CSS继承性是指自子标签是否能继承父标签的所有样式，并可以在父标签样式的基础上进行修改。利用继承关系可以大大缩小代码编写量，并提高可读性。

CSS层叠性可以理解为CSS解决‘’冲突“的方式。

标签样式冲突时会根据优先级使用优先级高的样式，相同优先级后定义样式会覆盖先定义样式。

a

a:link 代表未访问的链接
a:visited 访问过后的链接(判断是否访问过的依据是浏览器的浏览记录)
a:hover 代表鼠标悬停的链接
a:active 代表激活的链接

a标签链接到的资源 如果浏览器认识，那么浏览器会直接打开 如果不认识，则会下载。

元素不可见方法

  • display: none;

    元素不可见，不占据位置，在文档流中删除。

  • visbility: hidden;

    元素不可见，但是仍占据位置，存在在文档流中。子元素继承不可见属性，若要显示子元素显示声明可见。
