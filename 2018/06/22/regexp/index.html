javascript中的正则表达式

发表于 2018-06-22 | 分类于 javaScript |

ECMAScript通过RegExp数据类型来支持正则表达式。

字面量形式创建

var expression = / pattern / flags ;

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = /.at/gi;

  • pattern：

    任何简单或复杂的正则表达式。可包含字符类，限定符，分组，向前查找以及反向引用。

  • flags：标明正则表达式的行为，支持三个标志。
    • g：表示全局模式，即模式将被应用于所有字符串，而非发现第一个匹配时立即停止。
    • i：表示不区分大小写模式。
    • m：表示多行模式，即在一行文本末尾还会继续查找下一行是否存在与模式匹配的项。

使用RegExp构造函数创建

接收两个参数，第一个要匹配的字符串模式，第二个标志字符串。

var expression = new RegExp( "pattern", "flags" );

匹配字符串中所有以“at“结尾的三个字符的组合，不区分大小写。

var exp1 = new RegExp(".at", "gi");

注意：双转义

使用构造函数传入必须为字符串，在某些情况下要对字符进行双重转义。

下面给出部分例子：

字面量模式 等价的字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/age/ "name\\\/age"
/\d.\d{1,2}/ "\\d.\\d{1,2}"
/\w\\hello\\123/ "\\w\\\\hello\\\\123"

在ES3中，正则表达式字面量始终为共享同一个RegExp实例，而使用构造函数创建的每一个新RegExp都是一个新的实例。

在ES5中，明确规定字面量模式也必须创建新的实例。IE9，Firefox4+，Chrome做出了修改。

常用正则表达基本概念

pattern可包含内容：

  • 普通字符：没有显示指定为元字符的所有可打印不可打印字符。

    • 非打印字符，需要进行转义。

  • 特殊字符

    具有特殊含义的字符，包括元字符。需要进行转义。

    使用\进行转义，例如我们需要匹配a?b则应该写成var exp = /a\?b/g;

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | \d | 匹配一个数字字符，等价于[0-9] |
    | \D | 匹配一个非数字字符，等价于[^0-9] |
    | \s | 匹配任何空白字符，包括空格，制表符，换页符等。等价于[\f\n\r\t\v] |
    | \S | 匹配任何非空白字符。等价于[^\f\n\r\t\v] |
    | \w | 匹配字母，数字，下划线。等价于[A-Za-z0-9_] |
    | \w | 匹配非字母，数字，下划线。等价于[^A-Za-z0-9_] |

  • 限定符

    指定正则表达式中一个给定组件出现的次数。

    | 字符 | 描述 |
    | —– | —————– |
    | * | 0或多次 |
    | + | 1或多次 |
    | ？ | 0或1 |
    | {n} | n次 |
    | {n,} | 大于等于n次 |
    | {n,m} | n-m次（包含n，m） |

    贪心模式和非贪心模式转换

    *,+都是贪心模式，尽可能匹配多的字符串，当在*,+后面放置?可以将贪心模式转换为非贪心模式（最小匹配）。

  • 定位符

    | 字符 | 描述 |
    | —- | ———————————————————— |
    | ^ | 匹配输入字符串开始的位置。如果设置了m模式，还会与\n,\r之后的位置匹配。 |
    | $ | 匹配输入字符串结束的位置。如果设置了m模式，还会与\n,\r之前的位置匹配。 |
    | \b | 匹配一个字边界，即字与空格间的位置。 |
    | \B | 非字边界匹配。 |

  • 选择

    (选择1 | 选择2 | 选择3)

    圆括号副作用会使相关匹配被缓存，可使用?:消除缓存?:(选择1 | 选择2 | 选择3)

运算符优先级

从左到右进行计算，训训有限即规则

RegExp实例属性

RegExp每个实例都具有以下属性，可以获得有关模式信息

  • global：布尔值，是否设置g
  • ignoreCase：布尔值，是否设置i
  • lastIndex：整数，表示开始搜索下一个匹配项的字符位置，从0起算
  • multiline：布尔值，是否设置m
  • source：正则表达式的字符串表示，按照字面量形式字符串返回（即使使用构造函数创建）。

RegExp实例方法

exec()

  • 参数（1个）：要应用模式的字符串。

  • 返回值（Array）：包含第一个匹配项信息的数组，没有匹配项情况下返回null。

    两个额外属性：index（匹配项在字符串中的位置），input（应用正则表达式的字符串）。

    第一项：与整个模式匹配的字符串，其它项是与模式中的捕获组匹配的字符串。

test()

  • 参数（1个）：要应用模式的字符串。
  • 输出（布尔值）：模式与输入参数匹配返回true，否则false

常被用于if语句中判断目标字符串是否与模式匹配的简单检测。

toString(),toLocaleString()

返回正则表达式的字面量。（标准形式）

valueOf()

返回正则表达式本身

支持正则表达式的String对象的方法

search

string.search(searchvalue);

用于检索字符串中指定的字字符串，或检索与正则表达式想匹配的字符串。

返回匹配开始的下标，若没有找到匹配字串，返回-1。

match

string.match(regexp);

在字符串內检索指定的值，返回一个数组存储匹配的结果（很大程度上取决于是否有全局标志g）

replace

string.replace(searchvalue, newvalue);

在字符串中使用一些字符替换另一些指定字符，或替换一个与正则表达式匹配的字符。

split

string.split(separator, limit);

  • separator: 可选，指定分割位置。
  • limit：可选，指定返回数组的最大长度。

常用增则表达式总结

菜鸟总结

数字类

  • 数字\^[0-9]*$\
  • 零或非零开头的数字\^(0|[1-9][0-9]*)$\
  • 非零开头的最多带两位小数的数字\^([1-9][0-9]*)+(\.[0-9]{1-2})$\
  • 带1-2位小数的正数或负数\^(\-|\+)?\d+(\.\d{1,2})\

字符类

  • 汉字\^[\u4e00-\u9fa5]{0,}$\
  • 英文和数字\[A-Za-z0-9]+\
  • 有数字，英文字母或者下划线组成的字符串\^\w+$\
  • 禁止输入含有~的字符\ ~\x22 \

特殊需求表达式

  • Email地址: \^\w+([-+.]\w+)\*@\w+([-.]\w+)*\.\w+([-.]\w+)*$\
  • 腾讯QQ号：\^[1-9][0-9]{4,}$\ (腾讯QQ号从10000开始)
  • 中国邮政编码：\^[1-9]\d{5}(?!\d)\ (中国邮政编码为6位数字)
  • 日期格式：\^\d{4}-\d{1,2}-\d{1,2}\
