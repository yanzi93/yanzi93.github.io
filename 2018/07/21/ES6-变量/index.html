ES6扩展—变量

let

ES6 新增了的声明变量方法。但是所声明的变量，只在let命令所在的代码块内有效。

特点：

  1. 存在块级作用域
  2. 未声明不得使用，不存在变量提升
  3. 不允许在同一作用域下重复声明
  4. 暂时性死区（TDZ,temporal dead zone）
  5. 使用let声明变量不会挂载载window身上。（使用var声明变量挂载在window上）

与var区别

  • let仅在代码块內有效，在代码块外使用js报错。

  • 未声明不得使用，不存在变量提升。

    对于var声明变量，在声明前使用会提升为全局变量。

    对于let声明变量，在声明前使用会报错。

    个人理解： 在预编译时，会对var，function在第一次读取时进行编译，然后在运行时进行赋值。

  • 暂时性死区

    暂时性死区的本质就是，只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到声明变量的那一行代码出现，才可以获取和使用该变量。

    • 在块作用域中，使用let声明的变量在let声明前不得使用，即使这个变量是一个全局变量。

      导致使用typeof不会绝对安全

      自相赋值报错

  • 不允许在同一个作用域下重复声明

let块级作用域

  • 外层变量不受内层变量影响

  • ES6 允许块级作用域的任意嵌套。

  • 块级作用域的出现，实际上使得获得广泛应用的立即执行函数表达式（IIFE）不再必要了。

  • 代码块內函数声明

    块级作用域内声明的函数类似于let，对作用域之外没有影响。

    ES6 的块级作用域允许声明函数的规则，只在使用大括号的情况下成立，如果没有使用大括号，就会报错。

    考虑到环境导致的行为差异太大，应该避免在块级作用域内声明函数。如果确实需要，也应该写成函数表达式，而不是函数声明语句。

应用：

循环体以及需要重新赋值的变量。

在循环中，j使用let声明所以在每次循环代码块內有效。i使用var声明，所以不存在块作用域，i指向全局作用域。

const

特点：

  1. 一旦声明，常量的值就不能改变。
  2. 一旦声明变量，就必须立即初始化，不能留到以后赋值
  3. 定义Object，定义后仍可修改属性
  4. const的作用域与let命令相同：只在声明所在的块级作用域内有效。
  5. const命令声明的常量也是不提升，同样存在暂时性死区，只能在声明的位置后面使用。
  6. const声明的常量，也与let一样不可重复声明。

声明一个只读的常量。一旦声明，常量的值就不能改变。

一旦声明变量，就必须立即初始化，不能留到以后赋值。

本质：const声明变量不能改变，实际上是对const声明变量保存的内容（基本数据类型为值，引用类型为地址引用）不得改动。

  • 对于基本数据类型值就保存在内存地址中，改变值就改变了内存地址，会报错。
  • 对于引用数据，变量中保存内存地址的引用，所以当内存地址中的内容发生改变，但是保存的内存引用不变时，不会报错且改变成功保存。
  • 如果想让一个引用数据类型具有常量的特性，使用Object.freeze,冻结后非严格模式下修改不起作用，严格模式报错。

应用：

常量 ，配置项，引用的组件，中间变量等。

变量解构赋值

按照一定的模式，从数组和对象中提取值，对变量进行赋值，称为解构。

解构规则

数组解构赋值

  1. 按照对应顺序进行解构赋值。
  2. 可以不完全解构，即只匹配一部分。
  3. 可以嵌套解构。
  4. 可以指定默认值，解构后当参数默认值严格等于undefined时，默认值才生效。
  5. 默认值也可以为表达式，但是表达式为惰性请求，只有当undefined时才会求取。
  6. 默认值可以解构赋值的其他变量，但是变量必须已声明。
  7. 等号右边缺少数值活着解构不成功否会返回undefined。
  8. 等号右边不是数组（或不可遍历结构）会报错。

对象解构赋值

  • 解构规则

数组解构按照数组元素的次序排序，对象解构则没有次序。

对象解构按照的规则可以简单解释为，按照属性名相同进行解构，

let { foo, bar } = { foo: "aaa", bar: "bbb" };

进行解构时，前面部分相当于按照下面进行

let { foo: foo, bar: bar } = { foo: "aaa", bar: "bbb" };

属性名相同的，我们对值进行复制。

简单说就是先找到同名属性，然后在付给对应变量。

  • 特点：
  1. 按照解构规则，先找到同名属性，然后在付给对应变量。
  2. 未找到等于undefined。
  3. 可以嵌套解构。
  4. 可以使用默认值。
  5. 数组可以按照下标对象解构。

字符串，数组和布尔值的解构赋值

只要等号右边不是对象或数组，就先将其转换为对象。

由于undefined，null无法转换为对象，所以对他们进行解构赋值会报错。

函数参数解构赋值

  1. 使用参数解构和直接传入参数无差别
  2. 可以使用默认值

模式解构中的圆括号

在模式解构中使用圆括号，容易造成歧义。

报错情况

  1. 变量声明语句等号左边不能使用圆括号。let [(a)] = [1];
  2. 函数参数（属于变量声明）不能使用圆括号。function f([(z)]) { return z; }
  3. 赋值语句等号左边模式部分不能使用圆括号。({ p: a }) = { p: 42 };

可以使用圆括号情况

  1. 赋值语句等号左边非模式部分可以使用。({ p: (d) } = {});
