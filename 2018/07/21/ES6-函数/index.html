ES6中函数

发表于 2018-07-21 | 分类于 javaScript

ES6扩展—函数

参数默认值

规则

  • 在ES6中允许直接为函数参数设置默认值。

    在ES5中不能直接为函数参数指定默认值，必须使用||运算进行赋值，并且传入参数是false需要特别判断。

  • 参数变量是默认声明，不能使用let或const再次声明。

  • 使用默认参数时，函数不能有同名参数。

  • 参数默认值是惰性求值，只有在使用时进行计算。

  • 可以和解构赋值默认结合使用

    注意：{}很重要，当没有传入参数时，默认解析{}

  • 参数默认值的位置应该是尾参数

    非为参数设置默认值，实际上无法省略。因为后面参数不能跳过该参数赋值。除非显示声明该参数是undefined。

  • 函数的length属性

    指定默认值后，函数的length属性将不计算该参数。

  • 函数参数作用域问题

    当参数设置默认值后，函数进行声明初始化会形成一个单独作用域，函数初始化结束后作用域消失。不设置参数默认值不会出现。

应用

  • 利用参数默认值，可以指定某一个参数不得省略，省略就会抛出错误。
  • 设置参数默认值为undefined表示参数可以省略。

rest参数

ES6引入rest参数，形式为…变量名，用于获取函数的多余参数。但是rest参数必须置于函数参数最后一个，后面不能在跟参数（否则报错）。

优点

  • 可以向函数传入任意数目的参数。

  • arguments是类数组非数组，不能直接使用数组的方法需要使用Array.prototype.slice.call先将其转换为数组。rest参数不參在转换，就是一个数组。

  • 使用函数length属性，不包括rest参数。

严格模式

从ES5开始，函数内部可以设定为严格模式。必须显示声明。

ES6直有函数使用了默认值，解构赋值，扩展运算符，那么函数内部就不能显示设定为严格模式，否则报错。

原因：严格模式设定后包括函数参数和函数内部。函数先执行参数在执行函数内部，如果显示声明严格模式，则函数参数执行时还未声明。

解决办法：方法一：全局声明严格模式；方法二：把带默认参数函数包含在一个无参数的立即执行函数里面。

name属性

返回该函数的函数名。

ES5匿名匿名函数函数表达式返回空字符串。ES6匿名函数函数表达式返回函数实际函数名。

  • Function构造函数返回的函数实例，name属性的值为anonymous。

(new Function).name // "anonymous"

  • bind返回的函数，name属性值会加上bound前缀。

(Math.sin.bind()).name; //"bound sin"

箭头函数

规则

  • 参数部分使用圆括号

  • 代码块部分使用大括号扩起来，并使用return返回。

    • 仅有一条语句可省略大括号，省略return也会返回该语句。

    • 返回为对象且仅一句，大括号不能省略

    • 仅一条语句，并且不让该语句返回则使用

      let fn = () => void doesNotReturn();

  • 可以和变量解构结合使用

  • 简化回调函数

    [1,2,3].map(x => x * x);

    var result = values.sort((a, b) => a - b);

    const headAndTail = (head, ...tail) => [head, tail];

注意

  • 函数体内部的this，时定义时所在的对象，而非使用时所在的对象。
    1. 箭头函数可以让setTimeout里面的this绑定定义时所在的作用域，而不是运行时所在的作用域。
    2. 箭头函数让this指向固定化。原因是：箭头函数内部没有this，所以只能指向外层代码块的this。
    3. 箭头函数内部没有this，arguments，super，new.target。指向外层函数对应变量。不能使用call，applay，bind。
  • 不可以当作构造函数使用。
  • 不可以使用arguments对象，可以使用rest代替。
  • 不可以使用yield命令，因为尖头函数不能作Generator函数。

双冒号运算符

双冒号的提出是为了用来取代call，apply，bind调用。

双冒号绑定绑定运算符::，双冒号左边是一个对象，右边是一个函数。该运算符会自动将左边的对象作为上下文执行环境（即this对象），绑定到右边的函数上面。

尾调用优化

尾调用（Tail Call）是函数式编程概念。指某个函数的最后一步是调用另一个函数。
